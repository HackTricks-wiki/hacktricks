## Вбудовування токенів

Після токенізації текстових даних наступним критичним кроком підготовки даних для навчання великих мовних моделей (LLMs), таких як GPT, є створення **вбудовувань токенів**. Вбудовування токенів перетворюють дискретні токени (наприклад, слова або субслова) у неперервні числові вектори, які модель може обробляти та з яких може навчатися. Це пояснення розбиває поняття вбудовувань токенів, їх ініціалізацію, використання та роль позиційних вбудовувань у покращенні розуміння послідовностей токенів моделлю.

> [!TIP]
> Мета цього третього етапу дуже проста: **Призначити кожному з попередніх токенів у словнику вектор бажаної розмірності для навчання моделі.** Кожне слово у словнику буде точкою в X-вимірному просторі.\
> Зверніть увагу, що спочатку положення кожного слова в просторі просто ініціалізується «випадково», і ці положення є параметрами, які навчаються (будуть покращуватися під час навчання).
>
> Більше того, під час вбудовування токенів **створюється ще один шар вбудовувань**, який у цьому випадку представляє **абсолютну позицію слова в навчальному реченні**. Таким чином слово на різних позиціях у реченні матиме різне представлення (значення).

### **Що таке вбудовування токенів?**

**Вбудовування токенів** — це числові представлення токенів у неперервному векторному просторі. Кожному токену в словнику відповідає унікальний вектор фіксованої розмірності. Ці вектори фіксують семантичну та синтаксичну інформацію про токени, дозволяючи моделі розуміти зв’язки та патерни в даних.

- **Розмір словника:** Загальна кількість унікальних токенів (наприклад, слова, субслова) у словнику моделі.
- **Розмірність векторів:** Кількість числових значень (вимірів) у векторі кожного токена. Вища розмірність може захоплювати більш тонку інформацію, але потребує більше обчислювальних ресурсів.

**Приклад:**

- **Розмір словника:** 6 токенів \[1, 2, 3, 4, 5, 6]
- **Розмірність векторів:** 3 (x, y, z)

### **Ініціалізація вбудовувань токенів**

На початку навчання вбудовування токенів зазвичай ініціалізуються невеликими випадковими значеннями. Ці початкові значення коригуються (доопрацьовуються) під час навчання, щоб краще відображати значення токенів на основі тренувальних даних.

**Приклад PyTorch:**
```python
import torch

# Set a random seed for reproducibility
torch.manual_seed(123)

# Create an embedding layer with 6 tokens and 3 dimensions
embedding_layer = torch.nn.Embedding(6, 3)

# Display the initial weights (embeddings)
print(embedding_layer.weight)
```

### **Як ембеддинги токенів працюють під час навчання**

Під час навчання кожен токен у вхідних даних перетворюється на відповідний вектор ембеддингу. Ці вектори потім використовуються в різних обчисленнях у моделі, таких як механізми уваги та шари нейронної мережі.

**Приклад сценарію:**

- **Розмір батчу:** 8 (кількість зразків, що обробляються одночасно)
- **Максимальна довжина послідовності:** 4 (кількість токенів у зразку)
- **Розмірність ембеддингу:** 256

**Структура даних:**

- Кожен батч представлено як 3D тензор з формою `(batch_size, max_length, embedding_dim)`.
- Для нашого прикладу форма буде `(8, 4, 256)`.

## **Позиційні вбудування: додавання контексту до вбудувань токенів**

Хоча вбудування токенів захоплюють значення окремих токенів, вони за замовчуванням не кодують позицію токенів у послідовності. Розуміння порядку токенів має вирішальне значення для розуміння мови. Саме тут вступають у гру **позиційні вбудування**.

### **Чому потрібні позиційні вбудування:**

- **Порядок токенів важливий:** У реченнях значення часто залежить від порядку слів. Наприклад, "The cat sat on the mat" vs. "The mat sat on the cat."
- **Обмеження вбудувань:** Без позиційної інформації модель розглядає токени як «мішок слів», ігноруючи їхню послідовність.

### **Типи позиційних вбудувань:**

1. **Абсолютні позиційні вбудування:**
- Призначають унікальний позиційний вектор для кожної позиції в послідовності.
- **Приклад:** Перший токен у будь-якій послідовності має одне й те саме позиційне вбудування, другий токен має інше, і так далі.
- **Used By:** OpenAI’s GPT models.
2. **Відносні позиційні вбудування:**
- Кодують відносну відстань між токенами замість їх абсолютних позицій.
- **Приклад:** Вказують, наскільки далеко один токен від іншого, незалежно від їхніх абсолютних позицій у послідовності.
- **Used By:** Models like Transformer-XL and some variants of BERT.

### **Як інтегруються позиційні вбудування:**

- **Однакові розмірності:** Позиційні вбудування мають таку ж розмірність, як і вбудування токенів.
- **Додавання:** Їх додають до вбудувань токенів, поєднуючи ідентичність токена з позиційною інформацією без збільшення загальної розмірності.

**Приклад додавання позиційних вбудувань:**

Припустимо, що вектор вбудування токена дорівнює `[0.5, -0.2, 0.1]` і його позиційний вектор вбудування — `[0.1, 0.3, -0.1]`. Комбіноване вбудування, що використовується моделлю, буде:
```css
Combined Embedding = Token Embedding + Positional Embedding
= [0.5 + 0.1, -0.2 + 0.3, 0.1 + (-0.1)]
= [0.6, 0.1, 0.0]
```
**Переваги позиційних ембеддінгів:**

- **Усвідомлення контексту:** Модель може розрізняти токени залежно від їхніх позицій.
- **Розуміння послідовності:** Дозволяє моделі розуміти граматику, синтаксис та значення, залежні від контексту.

## **Позиційні ембеддінги в сучасних LLMs**

### **Ротаційні позиційні ембеддінги (RoPE)**

RoPE кодує позицію, застосовуючи залежне від позиції обертання до пар вимірів у векторах запитів/ключів, перетворюючи абсолютні позиції на відносні різниці фаз. Це надає інформацію про відносні позиції, не змінюючи розмірності ембеддінгів, і широко використовується в останніх decoder-only LLMs.

Детальніше про те, як токен- та позиційні ембеддінги комбінуються всередині моделі, див. [the LLM architecture page](5.-llm-architecture.md).

### **Розширення вікон контексту в моделях на основі RoPE**

Останні дослідження показують, що довжина контексту часто обмежується схемою позиційного кодування, а не самою матрицею токен-ембеддінгів.

- **Інтерполяція позицій (PI):** Перекалібровує індекси позицій так, щоб довші послідовності відображалися в діапазон, що зустрічався під час навчання, дозволяючи розширювати контекст з мінімальним донавчанням. Приклад:
```python
# Position Interpolation (PI) intuition
orig_ctx = 2048
new_ctx = 8192
scaled_pos = pos * (orig_ctx / new_ctx)
```
- **YaRN:** Обчислювально-ефективна стратегія розширення RoPE, яка модифікує масштабування/інтерполяцію RoPE для екстраполяції на довші контексти з меншою кількістю додаткових кроків навчання.

```python
BPE uses a vocabulary of 50257 words
Let's supose we want to use 256 dimensions (instead of the millions used by LLMs)
"""

vocab_size = 50257
output_dim = 256
token_embedding_layer = torch.nn.Embedding(vocab_size, output_dim)

## Generate the dataloader like before
max_length = 4
dataloader = create_dataloader_v1(
raw_text, batch_size=8, max_length=max_length,
stride=max_length, shuffle=False
)
data_iter = iter(dataloader)
inputs, targets = next(data_iter)

# Apply embeddings
token_embeddings = token_embedding_layer(inputs)
print(token_embeddings.shape)
torch.Size([8, 4, 256]) # 8 x 4 x 256

# Generate absolute embeddings
context_length = max_length
pos_embedding_layer = torch.nn.Embedding(context_length, output_dim)

pos_embeddings = pos_embedding_layer(torch.arange(max_length))

input_embeddings = token_embeddings + pos_embeddings
print(input_embeddings.shape) # torch.Size([8, 4, 256])
```
