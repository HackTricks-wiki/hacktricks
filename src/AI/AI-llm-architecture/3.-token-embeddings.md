## Token-inbeddings

Na die tokenisering van teksdata is die volgende kritieke stap in die voorbereiding van data vir die opleiding van groot taalmodelle (LLMs) soos GPT die skep van **token-inbeddings**. Token-inbeddings omskep diskrete tokens (soos woorde of subwoorde) in deurlopende numeriese vektore wat die model kan verwerk en waaruit hy kan leer. Hierdie verduideliking breek token-inbeddings af, hul initialisering, gebruik, en die rol van posisionele inbeddings in die verbetering van die model se begrip van tokenreekse.

> [!TIP]
> Die doel van hierdie derde fase is baie eenvoudig: **Ken aan elkeen van die vorige tokens in die woordeskat 'n vektor van die verlangde dimensies toe om die model op te lei.** Elke woord in die woordeskat sal 'n punt wees in 'n ruimte van X dimensies.\
> Let daarop dat aanvanklik die posisie van elke woord in die ruimte net "lukraak" geïnitialiseer word en hierdie posisies is trainbare parameters (sal tydens die opleiding verbeter word).
>
> Boonop word tydens die token-inbedding **nog 'n laag inbeddings geskep** wat (in hierdie geval) die **absolute posisie van die woord in die opleidingssin** verteenwoordig. Op hierdie manier sal 'n woord in verskillende posisies in die sin 'n ander voorstelling (betekenis) hê.

### **Wat is Token-inbeddings?**

**Token-inbeddings** is numeriese voorstellings van tokens in 'n deurlopende vektoruimte. Elke token in die woordeskat is geassosieer met 'n unieke vektor van vaste dimensies. Hierdie vektore vang semantiese en sintaktiese inligting van die tokens vas, wat die model in staat stel om verhoudings en patrone in die data te verstaan.

- **Woordeskatgrootte:** Die totale aantal unieke tokens (bv. woorde, subwoorde) in die model se woordeskat.
- **Inbeddingdimensies:** Die aantal numeriese waardes (dimensies) in elke token se vektor. Hoër dimensies kan meer genuanseerde inligting vasvang, maar vereis meer rekenaarhulpbronne.

**Voorbeeld:**

- **Woordeskatgrootte:** 6 tokens \[1, 2, 3, 4, 5, 6]
- **Inbeddingdimensies:** 3 (x, y, z)

### **Initialisering van Token-inbeddings**

Aan die begin van die opleiding word token-inbeddings tipies geïnitialiseer met klein lukrake waardes. Hierdie aanvanklike waardes word gedurende die opleiding aangepas (fyn afgestem) om die betekenisse van die tokens beter te verteenwoordig gebaseer op die opleidingsdata.

**PyTorch Example:**
```python
import torch

# Set a random seed for reproducibility
torch.manual_seed(123)

# Create an embedding layer with 6 tokens and 3 dimensions
embedding_layer = torch.nn.Embedding(6, 3)

# Display the initial weights (embeddings)
print(embedding_layer.weight)
```

### **Hoe Token Embeddings Werk Tydens Opleiding**

Tydens opleiding word elke token in die invoerdata omgeskakel na sy ooreenstemmende embedding-vektor. Hierdie vektore word dan in verskeie berekeninge binne die model gebruik, soos aandagmeganismes en neurale netwerklae.

**Voorbeeldsituasie:**

- **Batch Size:** 8 (aantal monsters wat gelyktydig verwerk word)
- **Max Sequence Length:** 4 (aantal tokens per monster)
- **Embedding Dimensions:** 256

**Data Structure:**

- Elke batch word voorgestel as 'n 3D-tensor met vorm `(batch_size, max_length, embedding_dim)`.
- Vir ons voorbeeld sou die vorm `(8, 4, 256)` wees.

## **Posisionele Inbeddings: Die byvoeging van konteks aan token-inbeddings**

Terwyl token-inbeddings die betekenis van individuele tokens vaslê, kodeer hulle nie noodwendig die posisie van tokens binne 'n volgorde nie. Om die volgorde van tokens te verstaan is van kardinale belang vir taalbegrip. Dit is waar **posisionele inbeddings** 'n rol speel.

### **Waarom posisionele inbeddings nodig is:**

- **Volgorde van Tokens is Belangrik:** In sinne hang die betekenis dikwels af van die volgorde van woorde. Byvoorbeeld, "Die kat het op die mat gesit" vs. "Die mat het op die kat gesit."
- **Beperkings van Inbeddings:** Sonder posisionele inligting behandel die model tokens as 'n "sak woorde", wat hul volgorde ignoreer.

### **Tipes posisionele inbeddings:**

1. **Absolute posisionele inbeddings:**
- Wys 'n unieke posisie-vektor toe aan elke posisie in die volgorde.
- **Voorbeeld:** Die eerste token in enige volgorde het dieselfde posisionele inbedding, die tweede token het 'n ander, ensovoorts.
- **Word gebruik deur:** OpenAI’s GPT models.
2. **Relatiewe posisionele inbeddings:**
- Kodeer die relatiewe afstand tussen tokens eerder as hul absolute posisies.
- **Voorbeeld:** Dui aan hoe ver twee tokens van mekaar is, ongeag hul absolute posisies in die volgorde.
- **Word gebruik deur:** Modelle soos Transformer-XL en sommige variante van BERT.

### **Hoe posisionele inbeddings geïntegreer word:**

- **Gelyke dimensies:** Posisionele inbeddings het dieselfde dimensionaliteit as token-inbeddings.
- **Optelling:** Hulle word by token-inbeddings gevoeg, wat token-identiteit met posisionele inligting kombineer sonder om die algehele dimensionaliteit te verhoog.

**Voorbeeld van die byvoeging van posisionele inbeddings:**

Gestel 'n token-inbedding-vektor is `[0.5, -0.2, 0.1]` en sy posisionele inbedding-vektor is `[0.1, 0.3, -0.1]`. Die gekombineerde inbedding wat deur die model gebruik word sou wees:
```css
Combined Embedding = Token Embedding + Positional Embedding
= [0.5 + 0.1, -0.2 + 0.3, 0.1 + (-0.1)]
= [0.6, 0.1, 0.0]
```
**Voordele van Posisionele embeddings:**

- **Konteksbewustheid:** Die model kan tussen tokens onderskei op grond van hul posisies.
- **Begrip van volgorde:** Maak dit vir die model moontlik om grammatika, sintaksis en konteksafhanklike betekenisse te verstaan.

## **Posisionele embeddings in Moderne LLMs**

### **Rotary Positional Embeddings (RoPE)**

RoPE kodeer posisie deur 'n posisie-afhanklike rotasie toe te pas op pare dimensies in die query/key-vektore, wat absolute posisies in relatiewe faseverskille omskakel. Dit verskaf relatiewe posisie-inligting terwyl die embedding-dimensionaliteit onveranderd bly en word wyd gebruik in onlangse decoder-only LLMs.

Vir hoe token en posisionele embeddings binne die model gekombineer word, sien [the LLM architecture page](5.-llm-architecture.md).

### **Uitbrei van konteksvensters in RoPE-gebaseerde modelle**

Onlangse werk toon dat kontekstlengte dikwels beperk word deur die posisionele kodering-skema eerder as deur die token embedding-matriks self.

- **Posisie-interpolasie (PI):** Skaleer posisie-indekse sodat langer sekwensies in die reeks val wat tydens training gesien is, wat uitbreiding met minimale fynafstemming moontlik maak. Voorbeeld:
```python
# Position Interpolation (PI) intuition
orig_ctx = 2048
new_ctx = 8192
scaled_pos = pos * (orig_ctx / new_ctx)
```
- **YaRN:** 'n rekenaardoeltreffende RoPE-uitbreidingsstrategie wat RoPE-skaal/interpolasie wysig om na langer kontekste te ekstrapoleer met minder bykomende opleidingsstappe.

# Create dimensional emdeddings

BPE uses a vocabulary of 50257 words
Let's supose we want to use 256 dimensions (instead of the millions used by LLMs)

vocab_size = 50257
output_dim = 256
token_embedding_layer = torch.nn.Embedding(vocab_size, output_dim)

## Generate the dataloader like before
max_length = 4
dataloader = create_dataloader_v1(
raw_text, batch_size=8, max_length=max_length,
stride=max_length, shuffle=False
)
data_iter = iter(dataloader)
inputs, targets = next(data_iter)

# Apply embeddings
token_embeddings = token_embedding_layer(inputs)
print(token_embeddings.shape)
torch.Size([8, 4, 256]) # 8 x 4 x 256

# Generate absolute embeddings
context_length = max_length
pos_embedding_layer = torch.nn.Embedding(context_length, output_dim)

pos_embeddings = pos_embedding_layer(torch.arange(max_length))

input_embeddings = token_embeddings + pos_embeddings
print(input_embeddings.shape) # torch.Size([8, 4, 256])
