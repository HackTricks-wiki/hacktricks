## Token-Einbettungen

Nach der Tokenisierung von Textdaten ist der nächste kritische Schritt bei der Vorbereitung von Daten zum Training großer Sprachmodelle (LLMs) wie GPT das Erstellen von Token-Einbettungen. Token-Einbettungen wandeln diskrete Token (wie Wörter oder Subwörter) in kontinuierliche numerische Vektoren um, die das Modell verarbeiten und aus denen es lernen kann. Diese Erklärung zerlegt Token-Einbettungen, ihre Initialisierung, Verwendung und die Rolle von positionellen Einbettungen bei der Verbesserung des Modellverständnisses von Token-Sequenzen.

> [!TIP]
> Das Ziel dieser dritten Phase ist sehr einfach: **Jedem der zuvor im Vokabular enthaltenen Tokens einen Vektor mit den gewünschten Dimensionen zuzuweisen, um das Modell zu trainieren.** Jedes Wort im Vokabular wird ein Punkt in einem Raum mit X Dimensionen sein.\
> Beachte, dass die Position jedes Wortes im Raum anfangs nur "zufällig" initialisiert ist und diese Positionen trainierbare Parameter sind (während des Trainings verbessert werden).
>
> Außerdem wird während der Token-Einbettung **eine weitere Ebene von Einbettungen erstellt**, die (in diesem Fall) die **absolute Position des Wortes im Trainingssatz** repräsentiert. Auf diese Weise hat ein Wort in unterschiedlichen Positionen im Satz eine unterschiedliche Repräsentation (Bedeutung).

### **Was sind Token-Einbettungen?**

**Token-Einbettungen** sind numerische Repräsentationen von Tokens in einem kontinuierlichen Vektorraum. Jedem Token im Vokabular ist ein eindeutiger Vektor mit fester Dimensionalität zugeordnet. Diese Vektoren erfassen semantische und syntaktische Informationen über die Tokens und ermöglichen es dem Modell, Beziehungen und Muster in den Daten zu erkennen.

- **Vokabulargröße:** Die Gesamtzahl der einzigartigen Tokens (z. B. Wörter, Subwörter) im Vokabular des Modells.
- **Embedding-Dimensionen:** Die Anzahl der numerischen Werte (Dimensionen) in jedem Token-Vektor. Höhere Dimensionen können nuanciertere Informationen erfassen, erfordern aber mehr Rechenressourcen.

**Beispiel:**

- **Vokabulargröße:** 6 Tokens \[1, 2, 3, 4, 5, 6]
- **Embedding-Dimensionen:** 3 (x, y, z)

### **Initialisierung von Token-Einbettungen**

Zu Beginn des Trainings werden Token-Einbettungen typischerweise mit kleinen Zufallswerten initialisiert. Diese Anfangswerte werden während des Trainings angepasst (feinabgestimmt), um die Bedeutungen der Tokens basierend auf den Trainingsdaten besser darzustellen.

**PyTorch-Beispiel:**
```python
import torch

# Set a random seed for reproducibility
torch.manual_seed(123)

# Create an embedding layer with 6 tokens and 3 dimensions
embedding_layer = torch.nn.Embedding(6, 3)

# Display the initial weights (embeddings)
print(embedding_layer.weight)
```

### **Wie Token-Embeddings während des Trainings funktionieren**

Während des Trainings wird jedes Token in den Eingabedaten in seinen entsprechenden Embedding-Vektor umgewandelt. Diese Vektoren werden dann in verschiedenen Berechnungen innerhalb des Modells verwendet, wie z. B. in Attention-Mechanismen und Schichten neuronaler Netze.

**Beispielszenario:**

- **Batch-Größe:** 8 (Anzahl der gleichzeitig verarbeiteten Samples)
- **Maximale Sequenzlänge:** 4 (Anzahl der Tokens pro Sample)
- **Embedding-Dimensionen:** 256

**Datenstruktur:**

- Jede Batch wird als 3D-Tensor mit der Form `(batch_size, max_length, embedding_dim)` dargestellt.
- Für unser Beispiel hätte die Form `(8, 4, 256)`.

## **Positional Embeddings: Adding Context to Token Embeddings**

Während Token-Embeddings die Bedeutung einzelner Token erfassen, kodieren sie nicht automatisch die Position der Token innerhalb einer Sequenz. Das Verständnis der Reihenfolge der Token ist für das Sprachverständnis entscheidend. Hier kommen **Positions-Embeddings** ins Spiel.

### **Why Positional Embeddings Are Needed:**

- **Token Order Matters:** In Sätzen hängt die Bedeutung oft von der Reihenfolge der Wörter ab. Zum Beispiel "The cat sat on the mat" vs. "The mat sat on the cat."
- **Embedding Limitation:** Ohne Positionsinformationen behandelt das Modell Token als eine "bag of words" und ignoriert ihre Reihenfolge.

### **Types of Positional Embeddings:**

1. **Absolute Positional Embeddings:**
- Weisen jeder Position in der Sequenz einen eindeutigen Positionsvektor zu.
- **Example:** Das erste Token in jeder Sequenz hat dasselbe Positions-Embedding, das zweite Token ein anderes, und so weiter.
- **Used By:** OpenAI’s GPT models.
2. **Relative Positional Embeddings:**
- Kodieren den relativen Abstand zwischen Token statt ihrer absoluten Positionen.
- **Example:** Geben an, wie weit zwei Token voneinander entfernt sind, unabhängig von ihren absoluten Positionen in der Sequenz.
- **Used By:** Models like Transformer-XL and some variants of BERT.

### **How Positional Embeddings Are Integrated:**

- **Same Dimensions:** Positions-Embeddings haben dieselbe Dimensionalität wie Token-Embeddings.
- **Addition:** Sie werden zu Token-Embeddings addiert und kombinieren Token-Identität mit Positionsinformationen, ohne die Gesamt-Dimensionalität zu erhöhen.

**Example of Adding Positional Embeddings:**

Angenommen, ein Token-Embedding-Vektor ist `[0.5, -0.2, 0.1]` und sein Positions-Embedding-Vektor ist `[0.1, 0.3, -0.1]`. Das kombinierte Embedding, das vom Modell verwendet wird, wäre:
```css
Combined Embedding = Token Embedding + Positional Embedding
= [0.5 + 0.1, -0.2 + 0.3, 0.1 + (-0.1)]
= [0.6, 0.1, 0.0]
```
**Vorteile von Positions-Einbettungen:**

- **Kontextbewusstsein:** Das Modell kann zwischen Tokens basierend auf deren Positionen unterscheiden.
- **Verständnis von Sequenzen:** Ermöglicht dem Modell, Grammatik, Syntax und kontextabhängige Bedeutungen zu erfassen.

## **Positions-Einbettungen in modernen LLMs**

### **Rotary Positional Embeddings (RoPE)**

RoPE codiert Positionen, indem eine positionsabhängige Rotation auf Paare von Dimensionen in den query/key vectors angewendet wird und dadurch absolute Positionen in relative Phasendifferenzen verwandelt werden. Das liefert relative Positionsinformationen, während die Dimensionalität der Embeddings unverändert bleibt, und wird in aktuellen decoder-only LLMs weit verbreitet eingesetzt.

Für die Art und Weise, wie token- und Positions-Einbettungen innerhalb des Modells kombiniert werden, siehe [the LLM architecture page](5.-llm-architecture.md).

### **Erweiterung von Kontextfenstern in RoPE-basierten Modellen**

Neuere Arbeiten zeigen, dass die Kontextlänge oft durch das Positionskodierungsschema begrenzt wird und nicht durch die Token-Einbettungsmatrix selbst.

- **Position Interpolation (PI):** Skaliert Positionsindizes so, dass längere Sequenzen in den während des Trainings gesehenen Bereich abgebildet werden, und ermöglicht so eine Erweiterung mit minimaler Feinabstimmung. Beispiel:
```python
# Position Interpolation (PI) intuition
orig_ctx = 2048
new_ctx = 8192
scaled_pos = pos * (orig_ctx / new_ctx)
```
- **YaRN:** Eine rechen-effiziente RoPE-Erweiterungsstrategie, die die RoPE-Skalierung/Interpolation verändert, um auf längere Kontexte zu extrapolieren und dafür weniger zusätzliche Trainingsschritte zu benötigen.

## Code-Beispiel

Im Folgenden das Code-Beispiel aus [https://github.com/rasbt/LLMs-from-scratch/blob/main/ch02/01_main-chapter-code/ch02.ipynb](https://github.com/rasbt/LLMs-from-scratch/blob/main/ch02/01_main-chapter-code/ch02.ipynb):
```python
# Use previous code...

# Create dimensional emdeddings
"""
BPE uses a vocabulary of 50257 words
Let's supose we want to use 256 dimensions (instead of the millions used by LLMs)
"""

vocab_size = 50257
output_dim = 256
token_embedding_layer = torch.nn.Embedding(vocab_size, output_dim)

## Generate the dataloader like before
max_length = 4
dataloader = create_dataloader_v1(
raw_text, batch_size=8, max_length=max_length,
stride=max_length, shuffle=False
)
data_iter = iter(dataloader)
inputs, targets = next(data_iter)

# Apply embeddings
token_embeddings = token_embedding_layer(inputs)
print(token_embeddings.shape)
torch.Size([8, 4, 256]) # 8 x 4 x 256

# Generate absolute embeddings
context_length = max_length
pos_embedding_layer = torch.nn.Embedding(context_length, output_dim)

pos_embeddings = pos_embedding_layer(torch.arange(max_length))

input_embeddings = token_embeddings + pos_embeddings
print(input_embeddings.shape) # torch.Size([8, 4, 256])
```
