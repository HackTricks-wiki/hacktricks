## Генерація тексту

Щоб навчити модель, потрібно, щоб вона могла генерувати нові токени. Потім ми порівнюватимемо згенеровані токени з очікуваними, щоб навчити модель **вивчати токени, які їй потрібно генерувати**.

Оскільки в попередніх прикладах ми вже передбачали деякі токени, цю функцію можна повторно використати для цієї мети.

> [!TIP]
> Мета цієї шостої фази дуже проста: **навчити модель з нуля**. Для цього буде використано попередню архітектуру LLM із циклами, які проходять через набори даних, використовуючи визначені функції втрат та optimizer для навчання всіх параметрів моделі.

## Оцінювання тексту

Для коректного навчання потрібно перевіряти передбачення, отримані для очікуваного токена. Мета навчання полягає в максимізації ймовірності правильного токена, що передбачає збільшення його ймовірності порівняно з іншими токенами.

Щоб максимізувати ймовірність правильного токена, ваги моделі потрібно змінити так, щоб ця ймовірність була максимальною. Оновлення ваг виконується за допомогою **backpropagation**. Для цього потрібна **функція втрат, яку потрібно максимізувати**. У цьому випадку функцією буде **різниця між виконаним передбаченням і бажаним результатом**.

Однак замість роботи із сирими передбаченнями використовуватиметься логарифм за основою n. Отже, якщо поточне передбачення очікуваного токена дорівнювало 7.4541e-05, натуральний логарифм (основа *e*) від **7.4541e-05** приблизно дорівнює **-9.5042**.\
Потім для кожного запису з контекстом довжиною, наприклад, 5 токенів моделі потрібно буде передбачити 5 токенів: першими 4 токенами будуть останні токени вхідних даних, а п'ятим — передбачений токен. Тому для кожного запису в цьому випадку буде 5 передбачень (навіть якщо перші 4 були у вхідних даних, модель цього не знає), а отже, 5 очікуваних токенів і 5 ймовірностей, які потрібно максимізувати.

Тому після застосування натурального логарифма до кожного передбачення обчислюється **середнє**, **знак мінус прибирається** (це називається _cross entropy loss_), і це **число потрібно зменшити якомога ближче до 0**, оскільки натуральний логарифм 1 дорівнює 0:

Інший спосіб виміряти якість моделі називається perplexity. **Perplexity** — це метрика, яка використовується для оцінювання того, наскільки добре ймовірнісна модель передбачає вибірку. У мовному моделюванні вона відображає **невпевненість моделі** під час передбачення наступного токена в послідовності.\
Наприклад, значення perplexity 48725 означає, що коли потрібно передбачити токен, модель не впевнена, який саме з 48 725 токенів у словнику є правильним.

```python
import tiktoken
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader


class GPTDatasetV1(Dataset):
def __init__(self, txt, tokenizer, max_length, stride):
self.input_ids = []
self.target_ids = []

# Tokenize the entire text
token_ids = tokenizer.encode(txt, allowed_special={"<|endoftext|>"})

# Use a sliding window to chunk the book into overlapping sequences of max_length
for i in range(0, len(token_ids) - max_length, stride):
input_chunk = token_ids[i:i + max_length]
target_chunk = token_ids[i + 1: i + max_length + 1]
self.input_ids.append(torch.tensor(input_chunk))
self.target_ids.append(torch.tensor(target_chunk))

def __len__(self):
return len(self.input_ids)

def __getitem__(self, idx):
return self.input_ids[idx], self.target_ids[idx]


def create_dataloader_v1(txt, batch_size=4, max_length=256,
stride=128, shuffle=True, drop_last=True, num_workers=0):
# Initialize the tokenizer
tokenizer = tiktoken.get_encoding("gpt2")

# Create dataset
dataset = GPTDatasetV1(txt, tokenizer, max_length, stride)

# Create dataloader
dataloader = DataLoader(
dataset, batch_size=batch_size, shuffle=shuffle, drop_last=drop_last, num_workers=num_workers)

return dataloader


class MultiHeadAttention(nn.Module):
def __init__(self, d_in, d_out, context_length, dropout, num_heads, qkv_bias=False):
super().__init__()
assert d_out % num_heads == 0, "d_out must be divisible by n_heads"

self.d_out = d_out
self.num_heads = num_heads
self.head_dim = d_out // num_heads  # Reduce the projection dim to match desired output dim

self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_key = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
self.out_proj = nn.Linear(d_out, d_out)  # Linear layer to combine head outputs
self.dropout = nn.Dropout(dropout)
self.register_buffer('mask', torch.triu(torch.ones(context_length, context_length), diagonal=1))

def forward(self, x):
b, num_tokens, d_in = x.shape

keys = self.W_key(x)  # Shape: (b, num_tokens, d_out)
queries = self.W_query(x)
values = self.W_value(x)

# We implicitly split the matrix by adding a `num_heads` dimension
# Unroll last dim: (b, num_tokens, d_out) -> (b, num_tokens, num_heads, head_dim)
keys = keys.view(b, num_tokens, self.num_heads, self.head_dim)
values = values.view(b, num_tokens, self.num_heads, self.head_dim)
queries = queries.view(b, num_tokens, self.num_heads, self.head_dim)

# Transpose: (b, num_tokens, num_heads, head_dim) -> (b, num_heads, num_tokens, head_dim)
keys = keys.transpose(1, 2)
queries = queries.transpose(1, 2)
values = values.transpose(1, 2)

# Compute scaled dot-product attention (aka self-attention) with a causal mask
attn_scores = queries @ keys.transpose(2, 3)  # Dot product for each head

# Original mask truncated to the number of tokens and converted to boolean
mask_bool = self.mask.bool()[:num_tokens, :num_tokens]

# Use the mask to fill attention scores
attn_scores.masked_fill_(mask_bool, -torch.inf)

attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
attn_weights = self.dropout(attn_weights)

# Shape: (b, num_tokens, num_heads, head_dim)
context_vec = (attn_weights @ values).transpose(1, 2)

# Combine heads, where self.d_out = self.num_heads * self.head_dim
context_vec = context_vec.reshape(b, num_tokens, self.d_out)
context_vec = self.out_proj(context_vec)  # optional projection

return context_vec


class LayerNorm(nn.Module):
def __init__(self, emb_dim):
super().__init__()
self.eps = 1e-5
self.scale = nn.Parameter(torch.ones(emb_dim))
self.shift = nn.Parameter(torch.zeros(emb_dim))

def forward(self, x):
mean = x.mean(dim=-1, keepdim=True)
var = x.var(dim=-1, keepdim=True, unbiased=False)
norm_x = (x - mean) / torch.sqrt(var + self.eps)
return self.scale * norm_x + self.shift


class GELU(nn.Module):
def __init__(self):
super().__init__()

def forward(self, x):
return 0.5 * x * (1 + torch.tanh(
torch.sqrt(torch.tensor(2.0 / torch.pi)) *
(x + 0.044715 * torch.pow(x, 3))
))


class FeedForward(nn.Module):
def __init__(self, cfg):
super().__init__()
self.layers = nn.Sequential(
nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),
GELU(),
nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"]),
)

def forward(self, x):
return self.layers(x)


class TransformerBlock(nn.Module):
def __init__(self, cfg):
super().__init__()
self.att = MultiHeadAttention(
d_in=cfg["emb_dim"],
d_out=cfg["emb_dim"],
context_length=cfg["context_length"],
num_heads=cfg["n_heads"],
dropout=cfg["drop_rate"],
qkv_bias=cfg["qkv_bias"])
self.ff = FeedForward(cfg)
self.norm1 = LayerNorm(cfg["emb_dim"])
self.norm2 = LayerNorm(cfg["emb_dim"])
self.drop_shortcut = nn.Dropout(cfg["drop_rate"])

def forward(self, x):
# Shortcut connection for attention block
shortcut = x
x = self.norm1(x)
x = self.att(x)   # Shape [batch_size, num_tokens, emb_size]
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

# Shortcut connection for feed-forward block
shortcut = x
x = self.norm2(x)
x = self.ff(x)
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

return x


class GPTModel(nn.Module):
def __init__(self, cfg):
super().__init__()
self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])
self.drop_emb = nn.Dropout(cfg["drop_rate"])

self.trf_blocks = nn.Sequential(
*[TransformerBlock(cfg) for _ in range(cfg["n_layers"])])

self.final_norm = LayerNorm(cfg["emb_dim"])
self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

def forward(self, in_idx):
batch_size, seq_len = in_idx.shape
tok_embeds = self.tok_emb(in_idx)
pos_embeds = self.pos_emb(torch.arange(seq_len, device=in_idx.device))
x = tok_embeds + pos_embeds  # Shape [batch_size, num_tokens, emb_size]
x = self.drop_emb(x)
x = self.trf_blocks(x)
x = self.final_norm(x)
logits = self.out_head(x)
return logits
```

### Функції генерації тексту

У попередньому розділі функція просто отримувала **найімовірніший токен** після отримання logits. Однак це означає, що для кожного введення завжди генеруватиметься однаковий результат, що робить його дуже детермінованим.

Наведена нижче функція `generate_text` застосовує концепції `top-k`, `temperature` і `multinomial`.

- **`top-k`** означає, що ми починаємо зводити до `-inf` усі ймовірності всіх токенів, крім k найімовірніших токенів. Отже, якщо k=3, перед прийняттям рішення лише 3 найімовірніші токени матимуть ймовірність, відмінну від `-inf`.
- **`temperature`** означає, що кожна ймовірність ділиться на значення temperature. Значення `0.1` збільшить перевагу найвищої ймовірності над найнижчою, тоді як temperature, наприклад, `5`, зробить розподіл більш пласким. Це допомагає збільшити варіативність відповідей, яку ми очікуємо від LLM.
- Після застосування temperature функція **`softmax`** застосовується знову, щоб сума ймовірностей усіх токенів, що залишилися, дорівнювала 1.
- Нарешті, замість вибору токена з найбільшою ймовірністю функція **`multinomial`** застосовується, щоб **передбачити наступний токен відповідно до кінцевих ймовірностей**. Отже, якщо токен 1 мав 70% ймовірності, токен 2 — 20%, а токен 3 — 10%, у 70% випадків буде вибрано токен 1, у 20% випадків — токен 2, а у 10% випадків — токен 3.
```python
# Generate text function
def generate_text(model, idx, max_new_tokens, context_size, temperature=0.0, top_k=None, eos_id=None):

# For-loop is the same as before: Get logits, and only focus on last time step
for _ in range(max_new_tokens):
idx_cond = idx[:, -context_size:]
with torch.no_grad():
logits = model(idx_cond)
logits = logits[:, -1, :]

# New: Filter logits with top_k sampling
if top_k is not None:
# Keep only top_k values
top_logits, _ = torch.topk(logits, top_k)
min_val = top_logits[:, -1]
logits = torch.where(logits < min_val, torch.tensor(float("-inf")).to(logits.device), logits)

# New: Apply temperature scaling
if temperature > 0.0:
logits = logits / temperature

# Apply softmax to get probabilities
probs = torch.softmax(logits, dim=-1)  # (batch_size, context_len)

# Sample from the distribution
idx_next = torch.multinomial(probs, num_samples=1)  # (batch_size, 1)

# Otherwise same as before: get idx of the vocab entry with the highest logits value
else:
idx_next = torch.argmax(logits, dim=-1, keepdim=True)  # (batch_size, 1)

if idx_next == eos_id:  # Stop generating early if end-of-sequence token is encountered and eos_id is specified
break

# Same as before: append sampled index to the running sequence
idx = torch.cat((idx, idx_next), dim=1)  # (batch_size, num_tokens+1)

return idx
```
> [!TIP]
> Існує поширена альтернатива `top-k`, яка називається [**`top-p`**](https://en.wikipedia.org/wiki/Top-p_sampling), також відома як nucleus sampling. Замість отримання k зразків із найвищою ймовірністю вона **впорядковує** весь **vocabulary** за ймовірностями та **підсумовує** їх від найвищої ймовірності до найнижчої, доки не буде **досягнуто порогового значення**.
>
> Потім **лише ці слова** з vocabulary враховуватимуться відповідно до їхніх відносних ймовірностей.
>
> Це усуває потребу вибирати кількість `k` зразків, оскільки оптимальне значення k може відрізнятися в кожному випадку, і достатньо вказати **лише порогове значення**.
>
> _Зверніть увагу, що це покращення не включене до наведеного вище коду._

> [!TIP]
> Ще один спосіб покращити згенерований текст — використовувати **Beam search** замість greedy search, застосованого в цьому прикладі.\
> На відміну від greedy search, який на кожному кроці вибирає наступне слово з найвищою ймовірністю та будує одну послідовність, **beam search відстежує 𝑘 k часткових послідовностей із найвищими оцінками** (так звані "beams") на кожному кроці. Досліджуючи кілька можливостей одночасно, цей підхід забезпечує баланс між ефективністю та якістю, підвищуючи ймовірність **знаходження кращої загальної** послідовності, яку greedy-підхід міг би пропустити через ранній вибір неоптимальних варіантів.
>
> _Зверніть увагу, що це покращення не включене до наведеного вище коду._

### Функції втрат

Функція **`calc_loss_batch`** обчислює cross entropy для прогнозу одного batch.\
Функція **`calc_loss_loader`** отримує cross entropy усіх batch і обчислює **середнє значення cross entropy**.
```python
# Define loss functions
def calc_loss_batch(input_batch, target_batch, model, device):
input_batch, target_batch = input_batch.to(device), target_batch.to(device)
logits = model(input_batch)
loss = torch.nn.functional.cross_entropy(logits.flatten(0, 1), target_batch.flatten())
return loss

def calc_loss_loader(data_loader, model, device, num_batches=None):
total_loss = 0.
if len(data_loader) == 0:
return float("nan")
elif num_batches is None:
num_batches = len(data_loader)
else:
# Reduce the number of batches to match the total number of batches in the data loader
# if num_batches exceeds the number of batches in the data loader
num_batches = min(num_batches, len(data_loader))
for i, (input_batch, target_batch) in enumerate(data_loader):
if i < num_batches:
loss = calc_loss_batch(input_batch, target_batch, model, device)
total_loss += loss.item()
else:
break
return total_loss / num_batches
```
> [!TIP]
> **Gradient clipping** — це техніка, що використовується для підвищення **стабільності навчання** великих нейронних мереж шляхом встановлення **максимального порогу** для величин градієнтів. Коли градієнти перевищують заздалегідь визначений `max_norm`, їх пропорційно зменшують, щоб оновлення параметрів моделі залишалися в керованому діапазоні, запобігаючи таким проблемам, як вибух градієнтів, і забезпечуючи більш контрольоване та стабільне навчання.

### Функції навчання

Функція `generate_and_print_sample` просто отримує контекст і генерує кілька токенів, щоб оцінити, наскільки добре модель працює на цьому етапі. Її викликає `train_model_simple` на кожному кроці.

Функція `evaluate_model` викликається з частотою, зазначеною у функції навчання, і використовується для вимірювання train loss та validation loss на цьому етапі навчання моделі.

Далі велика функція `train_model_simple` фактично навчає модель. Вона очікує:

- Train data loader (із даними, які вже розділені та підготовлені для навчання)
- Validator loader
- **optimizer**, який використовуватиметься під час навчання: це функція, що використовує градієнти та оновлює параметри для зменшення loss. У цьому випадку, як ви побачите, використовується `AdamW`, але існує багато інших оптимізаторів.
- `optimizer.zero_grad()` викликається в кожному циклі для скидання градієнтів, щоб вони не накопичувалися.
- Параметр **`lr`** — це **learning rate**, який визначає **розмір кроків**, що виконуються під час процесу оптимізації при оновленні параметрів моделі. **Менший** learning rate означає, що optimizer **виконує менші оновлення** ваг, що може забезпечити більш **точну** збіжність, але може **уповільнити** навчання. **Більший** learning rate може пришвидшити навчання, але **створює ризик перескочити** мінімум функції loss (**перестрибнути через** точку, у якій функція loss мінімізується).
- **Weight Decay** змінює етап **обчислення Loss**, додаючи додатковий член, який штрафує великі ваги. Це заохочує optimizer знаходити рішення з меншими вагами, забезпечуючи баланс між хорошим узгодженням із даними та простотою моделі, і запобігає overfitting у моделях машинного навчання, не дозволяючи моделі надавати надмірного значення окремій ознаці.
- Традиційні optimizers, як-от SGD із L2-регуляризацією, поєднують weight decay із градієнтом функції loss. Однак **AdamW** (варіант optimizer Adam) відокремлює weight decay від оновлення градієнта, що забезпечує ефективнішу регуляризацію.
- Пристрій, який використовуватиметься для навчання
- Кількість епох: кількість проходів по навчальних даних
- Частота оцінювання: частота виклику `evaluate_model`
- Ітерації оцінювання: кількість batch, які використовуються для оцінювання поточного стану моделі під час виклику `generate_and_print_sample`
- Початковий контекст: початкове речення, яке використовуватиметься під час виклику `generate_and_print_sample`
- Tokenizer
```python
# Functions to train the data
def train_model_simple(model, train_loader, val_loader, optimizer, device, num_epochs,
eval_freq, eval_iter, start_context, tokenizer):
# Initialize lists to track losses and tokens seen
train_losses, val_losses, track_tokens_seen = [], [], []
tokens_seen, global_step = 0, -1

# Main training loop
for epoch in range(num_epochs):
model.train()  # Set model to training mode

for input_batch, target_batch in train_loader:
optimizer.zero_grad() # Reset loss gradients from previous batch iteration
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss.backward() # Calculate loss gradients
optimizer.step() # Update model weights using loss gradients
tokens_seen += input_batch.numel()
global_step += 1

# Optional evaluation step
if global_step % eval_freq == 0:
train_loss, val_loss = evaluate_model(
model, train_loader, val_loader, device, eval_iter)
train_losses.append(train_loss)
val_losses.append(val_loss)
track_tokens_seen.append(tokens_seen)
print(f"Ep {epoch+1} (Step {global_step:06d}): "
f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")

# Print a sample text after each epoch
generate_and_print_sample(
model, tokenizer, device, start_context
)

return train_losses, val_losses, track_tokens_seen


def evaluate_model(model, train_loader, val_loader, device, eval_iter):
model.eval() # Set in eval mode to avoid dropout
with torch.no_grad():
train_loss = calc_loss_loader(train_loader, model, device, num_batches=eval_iter)
val_loss = calc_loss_loader(val_loader, model, device, num_batches=eval_iter)
model.train() # Back to training model applying all the configurations
return train_loss, val_loss


def generate_and_print_sample(model, tokenizer, device, start_context):
model.eval() # Set in eval mode to avoid dropout
context_size = model.pos_emb.weight.shape[0]
encoded = text_to_token_ids(start_context, tokenizer).to(device)
with torch.no_grad():
token_ids = generate_text(
model=model, idx=encoded,
max_new_tokens=50, context_size=context_size
)
decoded_text = token_ids_to_text(token_ids, tokenizer)
print(decoded_text.replace("\n", " "))  # Compact print format
model.train() # Back to training model applying all the configurations
```
> [!TIP]
> Щоб покращити швидкість навчання, існує кілька відповідних технік, які називаються **linear warmup** і **cosine decay.**
>
> **Linear warmup** полягає у визначенні початкової та максимальної швидкості навчання й її послідовному оновленні після кожної епохи. Це пояснюється тим, що початок навчання з меншими оновленнями ваг знижує ризик того, що модель зіткнеться з великими дестабілізуючими оновленнями під час фази навчання.\
> **Cosine decay** — це техніка, яка **поступово зменшує швидкість навчання** за напівкосинусоїдальною кривою **після** фази **warmup**, сповільнюючи оновлення ваг, щоб **мінімізувати ризик перевищення** мінімумів функції втрат і забезпечити стабільність навчання на пізніших етапах.
>
> _Зверніть увагу, що ці покращення не включені до наведеного вище коду._

```python
model.train(); # Put in training mode
```
Або просто модель, якщо ви плануєте лише її використовувати:
```python
# Save the model
torch.save(model.state_dict(), "model.pth")

# Load it
model = GPTModel(GPT_CONFIG_124M)

model.load_state_dict(torch.load("model.pth", map_location=device, weights_only=True))

model.eval() # Put in eval mode
```
### Практичні сучасні покращення навчання

Попереднього циклу достатньо, щоб зрозуміти механіку, але сучасне переднавчання зазвичай додає кілька оптимізацій для підвищення пропускної здатності та стабільності:

- **Змішана точність (`bfloat16` / `float16`)** для зменшення використання пам'яті та прискорення матричних множень. На практиці `bfloat16` зазвичай є кращим вибором на сучасних NVIDIA datacenter GPUs, оскільки зберігає ширший діапазон експоненти й зазвичай не потребує масштабування градієнтів, тоді як `float16` часто використовує `GradScaler`.
- **Накопичення градієнтів** для імітації більшого глобального розміру batch, коли весь batch не поміщається у VRAM.
- **Кліпінг градієнтів** (наприклад, `clip_grad_norm_`) для запобігання вибуховим оновленням, особливо на початку навчання.
- **Checkpointing активацій** для обміну додаткових обчислень на значно менше використання пам'яті шляхом повторного обчислення частини forward pass під час backpropagation.
- **Fused / memory-efficient attention** (`scaled_dot_product_attention`) для зменшення вартості attention під час роботи з довшими context windows.
- **Sharded training/checkpointing** (наприклад, FSDP / distributed checkpoints), коли модель або стани optimizer більше не поміщаються на одному GPU.

Мінімальний приклад, що поєднує деякі з цих підходів у PyTorch, виглядає так:
```python
accum_steps = 8
scaler = torch.amp.GradScaler("cuda")
optimizer.zero_grad(set_to_none=True)

for step, (input_batch, target_batch) in enumerate(train_loader, start=1):
with torch.autocast(device_type="cuda", dtype=torch.float16):
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss = loss / accum_steps

scaler.scale(loss).backward()

if step % accum_steps == 0:
scaler.unscale_(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
scaler.step(optimizer)
scaler.update()
optimizer.zero_grad(set_to_none=True)
```
> [!WARNING]
> Під час pre-training із checkpoint'ів або сторонніх репозиторіїв розглядайте **шлях завантаження як частину attack surface**. Віддавайте перевагу завантаженню `state_dict` або файлів `safetensors` замість цілих pickled Python objects, залишайте `trust_remote_code=False`, якщо ви не провели аудит репозиторію, і перевіряйте [../AI-Models-RCE.md](../AI-Models-RCE.md) щодо прикладів RCE під час завантаження моделей.

### Безпечніше та ефективніше за використанням пам'яті завантаження checkpoint'ів

Для великих checkpoint'ів наївна послідовність `torch.load(...)` + `load_state_dict(...)` може тимчасово дублювати пам'ять (ініціалізовані параметри + завантажені ваги). Новіші версії PyTorch додали кілька дуже корисних параметрів. Починаючи з PyTorch 2.6, `weights_only=True` є значенням за замовчуванням для `torch.load()`, якщо не передано власний pickle-модуль, але все одно варто явно вказувати його у прикладах:

- `weights_only=True`: обмежує десеріалізацію тензорами / примітивними об'єктами замість довільних pickled Python objects.
- `mmap=True`: відображає сховища тензорів у пам'ять безпосередньо з диска замість нетерплячого зчитування всього в RAM.
- `with torch.device("meta"):`: спочатку створює модуль із порожніми тензорами.
- `load_state_dict(..., assign=True)`: безпосередньо призначає завантажені тензори в модуль, ініціалізований через meta.
```python
state_dict = torch.load(
"model.pth",
map_location="cpu",
weights_only=True,
mmap=True,
)

with torch.device("meta"):
model = GPTModel(GPT_CONFIG_124M)

model.load_state_dict(state_dict, assign=True)
model.to(device)
model.eval()
```
Це особливо корисно під час завантаження чекпойнтів розміром у кілька ГБ на workstation, оскільки зменшує пікове споживання RAM під час запуску.

## Завантаження ваг GPT2

Є 2 швидкі скрипти для локального завантаження ваг GPT2. Для обох можна локально клонувати репозиторій [https://github.com/rasbt/LLMs-from-scratch](https://github.com/rasbt/LLMs-from-scratch), а потім:

- Скрипт [https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/01_main-chapter-code/gpt_generate.py](https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/01_main-chapter-code/gpt_generate.py) завантажить усі ваги та перетворить формати OpenAI у формати, очікувані нашим LLM. Скрипт також містить потрібну конфігурацію та prompt: "Every effort moves you"
- Скрипт [https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/02_alternative_weight_loading/weight-loading-hf-transformers.ipynb](https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/02_alternative_weight_loading/weight-loading-hf-transformers.ipynb) дає змогу локально завантажити будь-які ваги GPT2 (просто змініть змінну `CHOOSE_MODEL`) і генерувати текст на основі prompt.

Якщо замість GPT-2 потрібно завантажити сучасніший open-source чекпойнт із Hugging Face, зазвичай `transformers` автоматично розміщує модель, а за наявності слід надавати перевагу вагам `safetensors`:
```python
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

model_id = "google/gemma-7b"

tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(
model_id,
torch_dtype=torch.bfloat16,
device_map="auto",
low_cpu_mem_usage=True,
trust_remote_code=False,
)
```
`device_map="auto"` дуже зручний, коли повна checkpoint не вміщується в один accelerator, оскільки loader може розподіляти шари між пам’яттю GPU / CPU. Для великих repositories `save_pretrained(..., max_shard_size="5GB")` або еквівалентні Hub checkpoints створюють **sharded checkpoints** разом з індексом, завдяки чому їх можна завантажувати поступово, замість того щоб вимагати один величезний файл у RAM.

Якщо потрібно перевірити або розділити tensors без завантаження всього файлу, `safetensors` також підтримує часткове читання:
```python
from safetensors import safe_open

with safe_open("model.safetensors", framework="pt", device="cpu") as f:
emb_slice = f.get_slice("model.embed_tokens.weight")
vocab_size, hidden_dim = emb_slice.get_shape()
first_half = emb_slice[:, : hidden_dim // 2]
```
