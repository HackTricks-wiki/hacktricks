## Metin üretimi

Bir modeli eğitmek için modelin yeni token'lar üretebilmesini sağlamamız gerekir. Ardından, modeli **üretmesi gereken token'ları öğrenmesi** için eğitmek amacıyla üretilen token'ları beklenen token'larla karşılaştırırız.

Önceki örneklerde zaten bazı token'ları tahmin ettiğimiz için, bu işlevi yeniden kullanmak mümkündür.

> [!TIP]
> Bu altıncı aşamanın amacı oldukça basittir: **Modeli sıfırdan eğitmek**. Bunun için, modelin tüm parametrelerini eğitmek üzere tanımlanan loss function'lar ve optimizer'lar kullanılarak veri kümeleri üzerinde döngüler gerçekleştiren önceki LLM architecture kullanılacaktır.

## Metin değerlendirme

Doğru bir eğitim gerçekleştirmek için beklenen token için elde edilen tahminleri ölçüp kontrol etmek gerekir. Eğitimin amacı doğru token'ın olasılığını en üst düzeye çıkarmaktır; bu da diğer token'lara kıyasla olasılığını artırmayı içerir.

Doğru token'ın olasılığını en üst düzeye çıkarmak için modelin weights'leri, bu olasılığın en yüksek seviyeye çıkarılması amacıyla değiştirilmelidir. Weights güncellemeleri **backpropagation** aracılığıyla yapılır. Bunun için **en üst düzeye çıkarılacak bir loss function** gerekir. Bu durumda function, **gerçekleştirilen tahmin ile istenen tahmin arasındaki fark** olacaktır.

Ancak ham tahminlerle çalışmak yerine, tabanı n olan bir logarithm ile çalışılır. Dolayısıyla beklenen token için mevcut tahmin 7.4541e-05 ise, **7.4541e-05** sayısının doğal logarithm'ı (taban *e*) yaklaşık olarak **-9.5042** olur.\
Ardından, örneğin context length değeri 5 token olan her giriş için modelin 5 token tahmin etmesi gerekir; ilk 4 token input'un son token'ı, beşinci token ise tahmin edilen token olur. Bu nedenle, bu durumda her giriş için 5 tahmin elde ederiz (ilk 4 tanesi input'ta olsa bile model bunu bilmez); dolayısıyla 5 beklenen token ve en üst düzeye çıkarılacak 5 olasılık bulunur.

Bu nedenle, her tahmine doğal logarithm uygulandıktan sonra **average** hesaplanır, **minus symbol kaldırılır** (buna _cross entropy loss_ denir) ve bu, mümkün olduğunca 0'a yaklaştırılması gereken **sayıdır**; çünkü 1'in doğal logarithm'ı 0'dır:

```python
import tiktoken
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader


class GPTDatasetV1(Dataset):
def __init__(self, txt, tokenizer, max_length, stride):
self.input_ids = []
self.target_ids = []

# Tokenize the entire text
token_ids = tokenizer.encode(txt, allowed_special={"<|endoftext|>"})

# Use a sliding window to chunk the book into overlapping sequences of max_length
for i in range(0, len(token_ids) - max_length, stride):
input_chunk = token_ids[i:i + max_length]
target_chunk = token_ids[i + 1: i + max_length + 1]
self.input_ids.append(torch.tensor(input_chunk))
self.target_ids.append(torch.tensor(target_chunk))

def __len__(self):
return len(self.input_ids)

def __getitem__(self, idx):
return self.input_ids[idx], self.target_ids[idx]


def create_dataloader_v1(txt, batch_size=4, max_length=256,
stride=128, shuffle=True, drop_last=True, num_workers=0):
# Initialize the tokenizer
tokenizer = tiktoken.get_encoding("gpt2")

# Create dataset
dataset = GPTDatasetV1(txt, tokenizer, max_length, stride)

# Create dataloader
dataloader = DataLoader(
dataset, batch_size=batch_size, shuffle=shuffle, drop_last=drop_last, num_workers=num_workers)

return dataloader


class MultiHeadAttention(nn.Module):
def __init__(self, d_in, d_out, context_length, dropout, num_heads, qkv_bias=False):
super().__init__()
assert d_out % num_heads == 0, "d_out must be divisible by n_heads"

self.d_out = d_out
self.num_heads = num_heads
self.head_dim = d_out // num_heads  # Reduce the projection dim to match desired output dim

self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_key = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
self.out_proj = nn.Linear(d_out, d_out)  # Linear layer to combine head outputs
self.dropout = nn.Dropout(dropout)
self.register_buffer('mask', torch.triu(torch.ones(context_length, context_length), diagonal=1))

def forward(self, x):
b, num_tokens, d_in = x.shape

keys = self.W_key(x)  # Shape: (b, num_tokens, d_out)
queries = self.W_query(x)
values = self.W_value(x)

# We implicitly split the matrix by adding a `num_heads` dimension
# Unroll last dim: (b, num_tokens, d_out) -> (b, num_tokens, num_heads, head_dim)
keys = keys.view(b, num_tokens, self.num_heads, self.head_dim)
values = values.view(b, num_tokens, self.num_heads, self.head_dim)
queries = queries.view(b, num_tokens, self.num_heads, self.head_dim)

# Transpose: (b, num_tokens, num_heads, head_dim) -> (b, num_heads, num_tokens, head_dim)
keys = keys.transpose(1, 2)
queries = queries.transpose(1, 2)
values = values.transpose(1, 2)

# Compute scaled dot-product attention (aka self-attention) with a causal mask
attn_scores = queries @ keys.transpose(2, 3)  # Dot product for each head

# Original mask truncated to the number of tokens and converted to boolean
mask_bool = self.mask.bool()[:num_tokens, :num_tokens]

# Use the mask to fill attention scores
attn_scores.masked_fill_(mask_bool, -torch.inf)

attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
attn_weights = self.dropout(attn_weights)

# Shape: (b, num_tokens, num_heads, head_dim)
context_vec = (attn_weights @ values).transpose(1, 2)

# Combine heads, where self.d_out = self.num_heads * self.head_dim
context_vec = context_vec.reshape(b, num_tokens, self.d_out)
context_vec = self.out_proj(context_vec)  # optional projection

return context_vec


class LayerNorm(nn.Module):
def __init__(self, emb_dim):
super().__init__()
self.eps = 1e-5
self.scale = nn.Parameter(torch.ones(emb_dim))
self.shift = nn.Parameter(torch.zeros(emb_dim))

def forward(self, x):
mean = x.mean(dim=-1, keepdim=True)
var = x.var(dim=-1, keepdim=True, unbiased=False)
norm_x = (x - mean) / torch.sqrt(var + self.eps)
return self.scale * norm_x + self.shift


class GELU(nn.Module):
def __init__(self):
super().__init__()

def forward(self, x):
return 0.5 * x * (1 + torch.tanh(
torch.sqrt(torch.tensor(2.0 / torch.pi)) *
(x + 0.044715 * torch.pow(x, 3))
))


class FeedForward(nn.Module):
def __init__(self, cfg):
super().__init__()
self.layers = nn.Sequential(
nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),
GELU(),
nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"]),
)

def forward(self, x):
return self.layers(x)


class TransformerBlock(nn.Module):
def __init__(self, cfg):
super().__init__()
self.att = MultiHeadAttention(
d_in=cfg["emb_dim"],
d_out=cfg["emb_dim"],
context_length=cfg["context_length"],
num_heads=cfg["n_heads"],
dropout=cfg["drop_rate"],
qkv_bias=cfg["qkv_bias"])
self.ff = FeedForward(cfg)
self.norm1 = LayerNorm(cfg["emb_dim"])
self.norm2 = LayerNorm(cfg["emb_dim"])
self.drop_shortcut = nn.Dropout(cfg["drop_rate"])

def forward(self, x):
# Shortcut connection for attention block
shortcut = x
x = self.norm1(x)
x = self.att(x)   # Shape [batch_size, num_tokens, emb_size]
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

# Shortcut connection for feed-forward block
shortcut = x
x = self.norm2(x)
x = self.ff(x)
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

return x


class GPTModel(nn.Module):
def __init__(self, cfg):
super().__init__()
self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])
self.drop_emb = nn.Dropout(cfg["drop_rate"])

self.trf_blocks = nn.Sequential(
*[TransformerBlock(cfg) for _ in range(cfg["n_layers"])])

self.final_norm = LayerNorm(cfg["emb_dim"])
self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

def forward(self, in_idx):
batch_size, seq_len = in_idx.shape
tok_embeds = self.tok_emb(in_idx)
pos_embeds = self.pos_emb(torch.arange(seq_len, device=in_idx.device))
x = tok_embeds + pos_embeds  # Shape [batch_size, num_tokens, emb_size]
x = self.drop_emb(x)
x = self.trf_blocks(x)
x = self.final_norm(x)
logits = self.out_head(x)
return logits
```

### Metin oluşturma işlevleri

Önceki bir bölümde, logits değerlerini aldıktan sonra yalnızca **en olası token'ı** seçen bir işlev ele alınmıştı. Ancak bu, her girdi için her zaman aynı çıktının oluşturulacağı ve bunun da süreci çok deterministik hâle getireceği anlamına gelir.

Aşağıdaki `generate_text` işlevi, `top-k`, `temperature` ve `multinomial` kavramlarını uygular.

- **`top-k`**, en yüksek k token dışındaki tüm token'ların olasılıklarını `-inf` değerine düşürmeye başlayacağımız anlamına gelir. Örneğin k=3 ise bir karar verilmeden önce yalnızca en olası 3 token'ın `-inf` değerinden farklı bir olasılığı olur.
- **`temperature`**, her olasılığın temperature değerine bölüneceği anlamına gelir. `0.1` değeri, en yüksek olasılığı en düşük olasılığa kıyasla artırırken, örneğin `5` temperature değeri dağılımı daha düz hâle getirir. Bu, LLM'in yanıtlarında istediğimiz çeşitliliği artırmaya yardımcı olur.
- Temperature uygulandıktan sonra, kalan tüm token'ların toplam olasılığını 1 yapmak için yeniden bir **`softmax`** işlevi uygulanır.
- Son olarak, en yüksek olasılığa sahip token'ı seçmek yerine, **son olasılıklara göre bir sonraki token'ı tahmin etmek** için **`multinomial`** işlevi uygulanır. Örneğin token 1'in olasılığı %70, token 2'nin %20 ve token 3'ün %10 ise, zamanın %70'inde token 1, %20'sinde token 2 ve %10'unda token 3 seçilir.

```python
# Generate text function
def generate_text(model, idx, max_new_tokens, context_size, temperature=0.0, top_k=None, eos_id=None):

# For-loop is the same as before: Get logits, and only focus on last time step
for _ in range(max_new_tokens):
idx_cond = idx[:, -context_size:]
with torch.no_grad():
logits = model(idx_cond)
logits = logits[:, -1, :]

# New: Filter logits with top_k sampling
if top_k is not None:
# Keep only top_k values
top_logits, _ = torch.topk(logits, top_k)
min_val = top_logits[:, -1]
logits = torch.where(logits < min_val, torch.tensor(float("-inf")).to(logits.device), logits)

# New: Apply temperature scaling
if temperature > 0.0:
logits = logits / temperature

# Apply softmax to get probabilities
probs = torch.softmax(logits, dim=-1)  # (batch_size, context_len)

# Sample from the distribution
idx_next = torch.multinomial(probs, num_samples=1)  # (batch_size, 1)

# Otherwise same as before: get idx of the vocab entry with the highest logits value
else:
idx_next = torch.argmax(logits, dim=-1, keepdim=True)  # (batch_size, 1)

if idx_next == eos_id:  # Stop generating early if end-of-sequence token is encountered and eos_id is specified
break

# Same as before: append sampled index to the running sequence
idx = torch.cat((idx, idx_next), dim=1)  # (batch_size, num_tokens+1)

return idx
```
> [!TIP]
> `top-k` için yaygın bir alternatif, nucleus sampling olarak da bilinen [**`top-p`**](https://en.wikipedia.org/wiki/Top-p_sampling)'dir. Bu yöntem, en yüksek olasılığa sahip k örneği almak yerine, ortaya çıkan tüm **vocabulary** değerlerini olasılıklarına göre **düzenler** ve bir **eşik değerine ulaşılana** kadar bunları en yüksek olasılıktan en düşüğe doğru **toplar**.
>
> Ardından, vocabulary içindeki **yalnızca bu kelimeler**, göreli olasılıklarına göre dikkate alınır.
>
> Bu, `k` örneklerinin sayısını seçme gereksinimini ortadan kaldırır; çünkü en uygun k değeri her durumda farklı olabilir. Bunun yerine yalnızca bir **eşik değeri** belirlenir.
>
> _Bu iyileştirmenin önceki kodda bulunmadığını unutmayın._

> [!TIP]
> Üretilen metni iyileştirmenin başka bir yolu, bu örnekte kullanılan greedy search yerine **Beam search** kullanmaktır.\
> Her adımda en olası sonraki kelimeyi seçip tek bir sequence oluşturan greedy search'ün aksine, **beam search her adımda en yüksek puana sahip 𝑘 k kısmi sequence'i** (""beam"" olarak adlandırılır) takip eder. Birden fazla olasılığı eş zamanlı olarak keşfederek verimlilik ve kalite arasında denge kurar ve greedy yaklaşımın erken ve optimal olmayan seçimler nedeniyle gözden kaçırabileceği **daha iyi bir genel** sequence'i bulma olasılığını artırır.
>
> _Bu iyileştirmenin önceki kodda bulunmadığını unutmayın._

### Loss functions

**`calc_loss_batch`** function, tek bir batch'teki prediction'ın cross entropy değerini hesaplar.\
**`calc_loss_loader`**, tüm batch'lerin cross entropy değerini alır ve **ortalama cross entropy** değerini hesaplar.
```python
# Define loss functions
def calc_loss_batch(input_batch, target_batch, model, device):
input_batch, target_batch = input_batch.to(device), target_batch.to(device)
logits = model(input_batch)
loss = torch.nn.functional.cross_entropy(logits.flatten(0, 1), target_batch.flatten())
return loss

def calc_loss_loader(data_loader, model, device, num_batches=None):
total_loss = 0.
if len(data_loader) == 0:
return float("nan")
elif num_batches is None:
num_batches = len(data_loader)
else:
# Reduce the number of batches to match the total number of batches in the data loader
# if num_batches exceeds the number of batches in the data loader
num_batches = min(num_batches, len(data_loader))
for i, (input_batch, target_batch) in enumerate(data_loader):
if i < num_batches:
loss = calc_loss_batch(input_batch, target_batch, model, device)
total_loss += loss.item()
else:
break
return total_loss / num_batches
```
> [!TIP]
> **Gradient clipping**, gradient büyüklükleri için bir **maksimum eşik** belirleyerek büyük neural network'lerde **training stability** geliştirmek için kullanılan bir tekniktir. Gradient'ler önceden belirlenmiş `max_norm` değerini aştığında, model parametrelerine yapılan güncellemelerin yönetilebilir bir aralıkta kalmasını sağlamak için orantılı olarak küçültülür. Böylece exploding gradients gibi sorunlar önlenir ve daha kontrollü ve stabil bir training süreci sağlanır.
>
> _Bu iyileştirmenin önceki koda dahil edilmediğini unutmayın._

### Eğitim fonksiyonları

`generate_and_print_sample` fonksiyonu yalnızca bir context alır ve modelin o noktada ne kadar iyi olduğu hakkında fikir edinmek için bazı token'lar üretir. Bu fonksiyon her adımda `train_model_simple` tarafından çağrılır.

`evaluate_model` fonksiyonu, eğitim fonksiyonunda belirtilen sıklıkta çağrılır ve model eğitiminin o noktasındaki train loss ile validation loss değerlerini ölçmek için kullanılır.

Asıl modeli eğiten büyük fonksiyon ise `train_model_simple` fonksiyonudur. Bu fonksiyon şunları bekler:

- Train data loader (veriler zaten ayrılmış ve eğitim için hazırlanmış olmalıdır)
- Validator loader
- Eğitim sırasında kullanılacak **optimizer**: Gradient'leri kullanarak loss değerini azaltmak için parametreleri güncelleyen fonksiyondur. Bu durumda göreceğiniz üzere `AdamW` kullanılır, ancak daha birçok optimizer vardır.
- Her turda gradient'leri sıfırlamak ve bunların birikmesini önlemek için `optimizer.zero_grad()` çağrılır.
- **`lr`** parametresi, modelin parametreleri güncellenirken optimization process sırasında atılan **adımların boyutunu** belirleyen **learning rate** değeridir. **Daha küçük** bir learning rate, optimizer'ın ağırlıklarda **daha küçük güncellemeler** yapması anlamına gelir; bu, daha **hassas** bir convergence sağlayabilir ancak eğitimi **yavaşlatabilir**. **Daha büyük** bir learning rate eğitimi hızlandırabilir ancak loss function'ın minimum noktasını **aşma** (**loss function'ın minimize edildiği noktayı atlama**) riski taşır.
- **Weight Decay**, büyük ağırlıkları cezalandıran ek bir terim ekleyerek **Loss Calculation** adımını değiştirir. Bu, optimizer'ı daha küçük ağırlıklara sahip çözümler bulmaya teşvik eder; veriye iyi fit olma ile modeli basit tutma arasında denge kurarak, modelin herhangi bir özelliğe gereğinden fazla önem vermesini engeller ve machine learning modellerinde overfitting'i önlemeye yardımcı olur.
- SGD gibi L2 regularization kullanan geleneksel optimizer'lar, weight decay'i loss function'ın gradient'iyle birleştirir. Ancak **AdamW** (Adam optimizer'ın bir varyantı), weight decay'i gradient update işleminden ayırarak daha etkili regularization sağlar.
- Eğitim için kullanılacak device
- Epoch sayısı: Training data'nın üzerinden kaç kez geçileceği
- Evaluation frequency: `evaluate_model` fonksiyonunun çağrılma sıklığı
- Evaluation iteration: `generate_and_print_sample` çağrılırken modelin mevcut durumunu değerlendirmek için kullanılacak batch sayısı
- Start context: `generate_and_print_sample` çağrılırken kullanılacak başlangıç cümlesi
- Tokenizer
```python
# Functions to train the data
def train_model_simple(model, train_loader, val_loader, optimizer, device, num_epochs,
eval_freq, eval_iter, start_context, tokenizer):
# Initialize lists to track losses and tokens seen
train_losses, val_losses, track_tokens_seen = [], [], []
tokens_seen, global_step = 0, -1

# Main training loop
for epoch in range(num_epochs):
model.train()  # Set model to training mode

for input_batch, target_batch in train_loader:
optimizer.zero_grad() # Reset loss gradients from previous batch iteration
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss.backward() # Calculate loss gradients
optimizer.step() # Update model weights using loss gradients
tokens_seen += input_batch.numel()
global_step += 1

# Optional evaluation step
if global_step % eval_freq == 0:
train_loss, val_loss = evaluate_model(
model, train_loader, val_loader, device, eval_iter)
train_losses.append(train_loss)
val_losses.append(val_loss)
track_tokens_seen.append(tokens_seen)
print(f"Ep {epoch+1} (Step {global_step:06d}): "
f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")

# Print a sample text after each epoch
generate_and_print_sample(
model, tokenizer, device, start_context
)

return train_losses, val_losses, track_tokens_seen


def evaluate_model(model, train_loader, val_loader, device, eval_iter):
model.eval() # Set in eval mode to avoid dropout
with torch.no_grad():
train_loss = calc_loss_loader(train_loader, model, device, num_batches=eval_iter)
val_loss = calc_loss_loader(val_loader, model, device, num_batches=eval_iter)
model.train() # Back to training model applying all the configurations
return train_loss, val_loss


def generate_and_print_sample(model, tokenizer, device, start_context):
model.eval() # Set in eval mode to avoid dropout
context_size = model.pos_emb.weight.shape[0]
encoded = text_to_token_ids(start_context, tokenizer).to(device)
with torch.no_grad():
token_ids = generate_text(
model=model, idx=encoded,
max_new_tokens=50, context_size=context_size
)
decoded_text = token_ids_to_text(token_ids, tokenizer)
print(decoded_text.replace("\n", " "))  # Compact print format
model.train() # Back to training model applying all the configurations
```
> [!TIP]
> Öğrenme oranını iyileştirmek için **linear warmup** ve **cosine decay** olarak adlandırılan birkaç ilgili teknik vardır.
>
> **Linear warmup**, başlangıç ve maksimum öğrenme oranlarının tanımlanmasını ve her epoch sonrasında tutarlı bir şekilde güncellenmesini içerir. Bunun nedeni, eğitime daha küçük ağırlık güncellemeleriyle başlamanın, modelin eğitim aşamasında büyük ve dengesizleştirici güncellemelerle karşılaşma riskini azaltmasıdır.\
> **Cosine decay**, **warmup** aşamasından **sonra**, yarım kosinüs eğrisini izleyerek **öğrenme oranını kademeli olarak azaltan** bir tekniktir. Ağırlık güncellemelerini yavaşlatarak **loss minimumlarını aşma riskini en aza indirir** ve sonraki aşamalarda eğitim kararlılığını sağlar.
>
> _Bu iyileştirmelerin önceki kodda yer almadığını unutmayın._

### Pratik modern training iyileştirmeleri

Önceki döngü mekanikleri anlamak için yeterlidir, ancak modern pre-training genellikle throughput ve stability değerlerini iyileştirmek için çeşitli optimizasyonlar ekler:

- **Mixed precision (`bfloat16` / `float16`)**, memory kullanımını azaltır ve matrix multiplication işlemlerini hızlandırır. Uygulamada `bfloat16`, daha geniş bir exponent aralığını koruduğu ve genellikle gradient scaling gerektirmediği için yeni NVIDIA datacenter GPU'larında çoğunlukla tercih edilir; `float16` ise yaygın olarak bir `GradScaler` kullanır.
- **Gradient accumulation**, full batch VRAM'e sığmadığında daha büyük bir global batch size simüle eder.
- **Gradient clipping** (örneğin `clip_grad_norm_`), özellikle training'in ilk aşamalarında exploding update'leri önler.
- **Activation checkpointing**, backpropagation sırasında forward pass'in bir bölümünü yeniden hesaplayarak daha düşük memory kullanımı karşılığında ek compute maliyeti oluşturur.
- **Fused / memory-efficient attention** (`scaled_dot_product_attention`), daha uzun context window'lar için attention maliyetini azaltır.
- Model veya optimizer state'leri tek bir GPU'ya sığmadığında **sharded training/checkpointing** (örneğin FSDP / distributed checkpoints) kullanılır.

Bu fikirlerden bazılarını birleştiren minimal bir PyTorch örneği şöyledir:
```python
accum_steps = 8
scaler = torch.amp.GradScaler("cuda")
optimizer.zero_grad(set_to_none=True)

for step, (input_batch, target_batch) in enumerate(train_loader, start=1):
with torch.autocast(device_type="cuda", dtype=torch.float16):
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss = loss / accum_steps

scaler.scale(loss).backward()

if step % accum_steps == 0:
scaler.unscale_(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
scaler.step(optimizer)
scaler.update()
optimizer.zero_grad(set_to_none=True)
```
> [!WARNING]
> Checkpoint'lerden veya üçüncü taraf repository'lerden pre-training yaparken **loading path'i attack surface'in bir parçası olarak değerlendirin**. Whole pickled Python objects yerine `state_dict` veya `safetensors` dosyalarını yüklemeyi tercih edin, repository'yi denetlemediğiniz sürece `trust_remote_code=False` olarak bırakın ve model-loading RCE örnekleri için [../AI-Models-RCE.md](../AI-Models-RCE.md) dosyasını kontrol edin.

### Daha güvenli ve memory-efficient checkpoint loading

Büyük checkpoint'ler için naif `torch.load(...)` + `load_state_dict(...)` sıralaması, belleği geçici olarak iki katına çıkarabilir (başlatılmış parametreler + yüklenen ağırlıklar). Daha yeni PyTorch sürümleri oldukça kullanışlı bazı seçenekler ekledi. PyTorch 2.6'dan itibaren, özel bir pickle modülü geçirilmediğinde `torch.load()` için `weights_only=True` varsayılandır; ancak örneklerde bunu açıkça belirtmek yine de iyi bir fikirdir:

- `weights_only=True`: Deserialization işlemini rastgele pickled Python objects yerine tensor'ler / primitive objects ile sınırlar.
- `mmap=True`: Tensor storage'larını her şeyi RAM'e eager olarak okumak yerine diskten memory-map eder.
- `with torch.device("meta"):`: Önce modülü boş tensor'lerle oluşturur.
- `load_state_dict(..., assign=True)`: Yüklenen tensor'leri meta-initialized modüle doğrudan atar.
```python
state_dict = torch.load(
"model.pth",
map_location="cpu",
weights_only=True,
mmap=True,
)

with torch.device("meta"):
model = GPTModel(GPT_CONFIG_124M)

model.load_state_dict(state_dict, assign=True)
model.to(device)
model.eval()
```
Bu, başlangıç sırasında RAM sıçramalarını azalttığı için multi-GB checkpoint'leri bir workstation'a yüklerken özellikle kullanışlıdır.

Script ayrıca gerekli yapılandırma ve `"Every effort moves you"` prompt'u ile hazırlanmıştır.
- [https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/02_alternative_weight_loading/weight-loading-hf-transformers.ipynb](https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/02_alternative_weight_loading/weight-loading-hf-transformers.ipynb) script'i GPT2 ağırlıklarından herhangi birini yerel olarak yüklemenize (yalnızca `CHOOSE_MODEL` var'ını değiştirin) ve bazı prompt'lardan metin tahmin etmenize olanak tanır.

GPT-2 yerine Hugging Face'ten daha modern bir open-source checkpoint yüklemek istiyorsanız genellikle `transformers`'ın modeli otomatik olarak yerleştirmesine izin vermek ve mevcut olduğunda `safetensors` ağırlıklarını tercih etmek gerekir:
```python
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

model_id = "google/gemma-7b"

tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(
model_id,
torch_dtype=torch.bfloat16,
device_map="auto",
low_cpu_mem_usage=True,
trust_remote_code=False,
)
```
`device_map="auto"`, tam checkpoint tek bir accelerator'a sığmadığında oldukça kullanışlıdır; çünkü loader katmanları GPU / CPU belleğine dağıtabilir. Büyük repository'ler için `save_pretrained(..., max_shard_size="5GB")` veya eşdeğer Hub checkpoint'leri, **sharded checkpoint'ler** ve bir index oluşturur. Böylece RAM'de tek bir dev dosya gerektirmek yerine incremental olarak yüklenebilirler.

Tüm dosyayı yüklemeden tensor'ları incelemeniz veya dilimlemeniz gerekiyorsa, `safetensors` partial read işlemlerini de destekler:
```python
from safetensors import safe_open

with safe_open("model.safetensors", framework="pt", device="cpu") as f:
emb_slice = f.get_slice("model.embed_tokens.weight")
vocab_size, hidden_dim = emb_slice.get_shape()
first_half = emb_slice[:, : hidden_dim // 2]
```
