## Uzalishaji wa Maandishi

Ili ku-train model, tutahitaji model hiyo iweze kuzalisha tokens mpya. Kisha tutalinganisha tokens zilizozalishwa na zile zinazotarajiwa ili ku-train model **ijifunze tokens inazohitaji kuzalisha**.

Kama ilivyo katika mifano iliyotangulia ambapo tayari tulitabiri baadhi ya tokens, inawezekana kutumia tena function hiyo kwa madhumuni haya.

> [!TIP]
> Lengo la awamu hii ya sita ni rahisi sana: **Ku-train model kutoka mwanzo**. Kwa hili, architecture ya LLM ya awali itatumika pamoja na loops zinazopitia data sets kwa kutumia loss functions na optimizer zilizofafanuliwa ili ku-train parameters zote za model.

## Tathmini ya Maandishi

Ili kufanya training sahihi, ni muhimu kupima na kukagua predictions zilizopatikana dhidi ya token inayotarajiwa. Lengo la training ni kuongeza uwezekano wa token sahihi, jambo linalohusisha kuongeza probability yake ikilinganishwa na tokens nyingine.

Ili kuongeza probability ya token sahihi, weights za model lazima zibadilishwe ili probability hiyo iongezwe. Updates za weights hufanywa kupitia **backpropagation**. Hii inahitaji **loss function ya kuongeza**. Katika hali hii, function hiyo itakuwa **tofauti kati ya prediction iliyofanywa na ile inayotakiwa**.

Hata hivyo, badala ya kufanya kazi na predictions ghafi, itatumia logarithm yenye base n. Kwa hivyo, ikiwa prediction ya sasa ya token inayotarajiwa ilikuwa 7.4541e-05, logarithm ya asili (base *e*) ya **7.4541e-05** ni takriban **-9.5042**.\
Kisha, kwa kila entry yenye context length ya tokens 5 kwa mfano, model itahitaji kutabiri tokens 5, ambapo tokens 4 za kwanza ni token ya mwisho ya input, na ya tano ndiyo iliyotabiriwa. Kwa hiyo, kwa kila entry tutakuwa na predictions 5 katika hali hiyo (hata kama 4 za kwanza zilikuwa kwenye input, model haijui hili), zikiwa na tokens 5 zinazotarajiwa na hivyo probabilities 5 za kuongeza.

Kwa hiyo, baada ya kutumia logarithm ya asili kwa kila prediction, **average** huhesabiwa, **minus symbol huondolewa** (hii huitwa _cross entropy loss_), na hiyo ndiyo **namba inayopaswa kupunguzwa iwe karibu na 0 iwezekanavyo**, kwa sababu logarithm ya asili ya 1 ni 0:

```python
import tiktoken
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader


class GPTDatasetV1(Dataset):
def __init__(self, txt, tokenizer, max_length, stride):
self.input_ids = []
self.target_ids = []

# Tokenize the entire text
token_ids = tokenizer.encode(txt, allowed_special={"<|endoftext|>"})

# Use a sliding window to chunk the book into overlapping sequences of max_length
for i in range(0, len(token_ids) - max_length, stride):
input_chunk = token_ids[i:i + max_length]
target_chunk = token_ids[i + 1: i + max_length + 1]
self.input_ids.append(torch.tensor(input_chunk))
self.target_ids.append(torch.tensor(target_chunk))

def __len__(self):
return len(self.input_ids)

def __getitem__(self, idx):
return self.input_ids[idx], self.target_ids[idx]


def create_dataloader_v1(txt, batch_size=4, max_length=256,
stride=128, shuffle=True, drop_last=True, num_workers=0):
# Initialize the tokenizer
tokenizer = tiktoken.get_encoding("gpt2")

# Create dataset
dataset = GPTDatasetV1(txt, tokenizer, max_length, stride)

# Create dataloader
dataloader = DataLoader(
dataset, batch_size=batch_size, shuffle=shuffle, drop_last=drop_last, num_workers=num_workers)

return dataloader


class MultiHeadAttention(nn.Module):
def __init__(self, d_in, d_out, context_length, dropout, num_heads, qkv_bias=False):
super().__init__()
assert d_out % num_heads == 0, "d_out must be divisible by n_heads"

self.d_out = d_out
self.num_heads = num_heads
self.head_dim = d_out // num_heads  # Reduce the projection dim to match desired output dim

self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_key = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
self.out_proj = nn.Linear(d_out, d_out)  # Linear layer to combine head outputs
self.dropout = nn.Dropout(dropout)
self.register_buffer('mask', torch.triu(torch.ones(context_length, context_length), diagonal=1))

def forward(self, x):
b, num_tokens, d_in = x.shape

keys = self.W_key(x)  # Shape: (b, num_tokens, d_out)
queries = self.W_query(x)
values = self.W_value(x)

# We implicitly split the matrix by adding a `num_heads` dimension
# Unroll last dim: (b, num_tokens, d_out) -> (b, num_tokens, num_heads, head_dim)
keys = keys.view(b, num_tokens, self.num_heads, self.head_dim)
values = values.view(b, num_tokens, self.num_heads, self.head_dim)
queries = queries.view(b, num_tokens, self.num_heads, self.head_dim)

# Transpose: (b, num_tokens, num_heads, head_dim) -> (b, num_heads, num_tokens, head_dim)
keys = keys.transpose(1, 2)
queries = queries.transpose(1, 2)
values = values.transpose(1, 2)

# Compute scaled dot-product attention (aka self-attention) with a causal mask
attn_scores = queries @ keys.transpose(2, 3)  # Dot product for each head

# Original mask truncated to the number of tokens and converted to boolean
mask_bool = self.mask.bool()[:num_tokens, :num_tokens]

# Use the mask to fill attention scores
attn_scores.masked_fill_(mask_bool, -torch.inf)

attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
attn_weights = self.dropout(attn_weights)

# Shape: (b, num_tokens, num_heads, head_dim)
context_vec = (attn_weights @ values).transpose(1, 2)

# Combine heads, where self.d_out = self.num_heads * self.head_dim
context_vec = context_vec.reshape(b, num_tokens, self.d_out)
context_vec = self.out_proj(context_vec)  # optional projection

return context_vec


class LayerNorm(nn.Module):
def __init__(self, emb_dim):
super().__init__()
self.eps = 1e-5
self.scale = nn.Parameter(torch.ones(emb_dim))
self.shift = nn.Parameter(torch.zeros(emb_dim))

def forward(self, x):
mean = x.mean(dim=-1, keepdim=True)
var = x.var(dim=-1, keepdim=True, unbiased=False)
norm_x = (x - mean) / torch.sqrt(var + self.eps)
return self.scale * norm_x + self.shift


class GELU(nn.Module):
def __init__(self):
super().__init__()

def forward(self, x):
return 0.5 * x * (1 + torch.tanh(
torch.sqrt(torch.tensor(2.0 / torch.pi)) *
(x + 0.044715 * torch.pow(x, 3))
))


class FeedForward(nn.Module):
def __init__(self, cfg):
super().__init__()
self.layers = nn.Sequential(
nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),
GELU(),
nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"]),
)

def forward(self, x):
return self.layers(x)


class TransformerBlock(nn.Module):
def __init__(self, cfg):
super().__init__()
self.att = MultiHeadAttention(
d_in=cfg["emb_dim"],
d_out=cfg["emb_dim"],
context_length=cfg["context_length"],
num_heads=cfg["n_heads"],
dropout=cfg["drop_rate"],
qkv_bias=cfg["qkv_bias"])
self.ff = FeedForward(cfg)
self.norm1 = LayerNorm(cfg["emb_dim"])
self.norm2 = LayerNorm(cfg["emb_dim"])
self.drop_shortcut = nn.Dropout(cfg["drop_rate"])

def forward(self, x):
# Shortcut connection for attention block
shortcut = x
x = self.norm1(x)
x = self.att(x)   # Shape [batch_size, num_tokens, emb_size]
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

# Shortcut connection for feed-forward block
shortcut = x
x = self.norm2(x)
x = self.ff(x)
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

return x


class GPTModel(nn.Module):
def __init__(self, cfg):
super().__init__()
self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])
self.drop_emb = nn.Dropout(cfg["drop_rate"])

self.trf_blocks = nn.Sequential(
*[TransformerBlock(cfg) for _ in range(cfg["n_layers"])])

self.final_norm = LayerNorm(cfg["emb_dim"])
self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

def forward(self, in_idx):
batch_size, seq_len = in_idx.shape
tok_embeds = self.tok_emb(in_idx)
pos_embeds = self.pos_emb(torch.arange(seq_len, device=in_idx.device))
x = tok_embeds + pos_embeds  # Shape [batch_size, num_tokens, emb_size]
x = self.drop_emb(x)
x = self.trf_blocks(x)
x = self.final_norm(x)
logits = self.out_head(x)
return logits
```

### Kazi za generate text

Katika sehemu iliyotangulia, kulikuwa na function iliyopata tu **token yenye uwezekano mkubwa zaidi** baada ya kupata logits. Hata hivyo, hii inamaanisha kuwa kwa kila ingizo, output ileile itazalishwa kila mara, jambo linaloifanya iwe ya deterministic sana.

Function ifuatayo ya `generate_text` itatumia dhana za `top-k`, `temperature` na `multinomial`.

- **`top-k`** inamaanisha kwamba tutaanza kupunguza hadi `-inf` probabilities zote za tokens isipokuwa tokens k bora. Kwa hiyo, ikiwa k=3, kabla ya kufanya uamuzi, ni tokens 3 tu zenye uwezekano mkubwa zaidi zitakuwa na probability tofauti na `-inf`.
- **`temperature`** inamaanisha kwamba kila probability itagawanywa kwa thamani ya temperature. Thamani ya `0.1` itaongeza tofauti ya probability ya juu ikilinganishwa na ile ya chini, huku temperature ya `5`, kwa mfano, itafanya probabilities ziwe sawia zaidi. Hii husaidia kuongeza variation katika majibu tunayotaka LLM iwe nayo.
- Baada ya kutumia temperature, function ya **`softmax`** itatumika tena ili tokens zote zinazosalia ziwe na jumla ya probability ya 1.
- Hatimaye, badala ya kuchagua token yenye probability kubwa zaidi, function ya **`multinomial`** itatumika ili **kutabiri token inayofuata kulingana na probabilities za mwisho**. Kwa hiyo, ikiwa token 1 ina probability ya 70%, token 2 ina 20% na token 3 ina 10%, token 1 itachaguliwa katika 70% ya mara, token 2 katika 20% ya mara, na token 3 katika 10% ya mara.

```python
# Generate text function
def generate_text(model, idx, max_new_tokens, context_size, temperature=0.0, top_k=None, eos_id=None):

# For-loop is the same as before: Get logits, and only focus on last time step
for _ in range(max_new_tokens):
idx_cond = idx[:, -context_size:]
with torch.no_grad():
logits = model(idx_cond)
logits = logits[:, -1, :]

# New: Filter logits with top_k sampling
if top_k is not None:
# Keep only top_k values
top_logits, _ = torch.topk(logits, top_k)
min_val = top_logits[:, -1]
logits = torch.where(logits < min_val, torch.tensor(float("-inf")).to(logits.device), logits)

# New: Apply temperature scaling
if temperature > 0.0:
logits = logits / temperature

# Apply softmax to get probabilities
probs = torch.softmax(logits, dim=-1)  # (batch_size, context_len)

# Sample from the distribution
idx_next = torch.multinomial(probs, num_samples=1)  # (batch_size, 1)

# Otherwise same as before: get idx of the vocab entry with the highest logits value
else:
idx_next = torch.argmax(logits, dim=-1, keepdim=True)  # (batch_size, 1)

if idx_next == eos_id:  # Stop generating early if end-of-sequence token is encountered and eos_id is specified
break

# Same as before: append sampled index to the running sequence
idx = torch.cat((idx, idx_next), dim=1)  # (batch_size, num_tokens+1)

return idx
```
> [!TIP]
> Kuna mbadala unaotumika sana wa `top-k` unaoitwa [**`top-p`**](https://en.wikipedia.org/wiki/Top-p_sampling), unaojulikana pia kama nucleus sampling, ambao badala ya kupata sampuli k zenye uwezekano mkubwa zaidi, **hupanga** **vocabulary** yote inayotokana kulingana na uwezekano na **huzi-jumlisha** kutoka uwezekano mkubwa zaidi hadi mdogo zaidi mpaka **kizingiti kifikiwe**.
>
> Kisha, **maneno hayo pekee** ya vocabulary yatazingatiwa kulingana na uwezekano wao wa kulinganisha.
>
> Hii inawezesha kutohitaji kuchagua idadi ya sampuli `k`, kwa sababu k inayofaa inaweza kuwa tofauti katika kila hali, bali kuchagua **kizingiti** pekee.
>
> _Kumbuka kuwa uboreshaji huu haujajumuishwa katika code ya awali._

> [!TIP]
> Njia nyingine ya kuboresha maandishi yanayotengenezwa ni kutumia **Beam search** badala ya greedy search iliyotumika katika mfano huu.\
> Tofauti na greedy search, ambayo huchagua neno linalofuata lenye uwezekano mkubwa zaidi katika kila hatua na kujenga mfuatano mmoja, **beam search hufuatilia mfuatano nusu kamili k wenye alama za juu zaidi** (unaoitwa "beams") katika kila hatua. Kwa kuchunguza uwezekano kadhaa kwa wakati mmoja, husawazisha ufanisi na ubora, na kuongeza uwezekano wa **kupata mfuatano bora zaidi kwa ujumla** ambao huenda ukapitwa na mbinu ya greedy kutokana na maamuzi ya mapema yasiyofaa.
>
> _Kumbuka kuwa uboreshaji huu haujajumuishwa katika code ya awali._

### Loss functions

Function ya **`calc_loss_batch`** huhesabu cross entropy ya utabiri wa batch moja.\
Function ya **`calc_loss_loader`** hupata cross entropy ya batches zote na huhesabu **wastani wa cross entropy**.
```python
# Define loss functions
def calc_loss_batch(input_batch, target_batch, model, device):
input_batch, target_batch = input_batch.to(device), target_batch.to(device)
logits = model(input_batch)
loss = torch.nn.functional.cross_entropy(logits.flatten(0, 1), target_batch.flatten())
return loss

def calc_loss_loader(data_loader, model, device, num_batches=None):
total_loss = 0.
if len(data_loader) == 0:
return float("nan")
elif num_batches is None:
num_batches = len(data_loader)
else:
# Reduce the number of batches to match the total number of batches in the data loader
# if num_batches exceeds the number of batches in the data loader
num_batches = min(num_batches, len(data_loader))
for i, (input_batch, target_batch) in enumerate(data_loader):
if i < num_batches:
loss = calc_loss_batch(input_batch, target_batch, model, device)
total_loss += loss.item()
else:
break
return total_loss / num_batches
```
> [!TIP]
> **Gradient clipping** ni mbinu inayotumiwa kuimarisha **training stability** katika neural networks kubwa kwa kuweka **maximum threshold** ya ukubwa wa gradients. Gradients zinapozidi `max_norm` iliyowekwa awali, hupunguzwa kwa uwiano ili kuhakikisha kwamba masasisho ya parameters za model yanabaki ndani ya kiwango kinachoweza kudhibitiwa, kuzuia matatizo kama exploding gradients na kuhakikisha training yenye udhibiti na uthabiti zaidi.
>
> _Kumbuka kwamba uboreshaji huu haujajumuishwa kwenye code ya awali._

### Kazi za training

Function `generate_and_print_sample` itapokea context na generate tokens kadhaa ili kupata hisia kuhusu ubora wa model katika hatua hiyo. Hii inaitwa na `train_model_simple` katika kila step.

Function `evaluate_model` inaitwa mara kwa mara kama ilivyoainishwa kwenye training function na hutumika kupima train loss na validation loss katika hatua hiyo ya training ya model.

Kisha function kubwa `train_model_simple` ndiyo inayofanya training ya model. Inatarajia:

- Train data loader (ikiwa na data ambayo tayari imetenganishwa na kuandaliwa kwa ajili ya training)
- Validator loader
- **optimizer** ya kutumia wakati wa training: Hii ndiyo function itakayotumia gradients na kusasisha parameters ili kupunguza loss. Katika hali hii, kama utakavyoona, `AdamW` inatumika, lakini kuna nyingine nyingi.
- `optimizer.zero_grad()` inaitwa ili kuweka upya gradients katika kila round na kuzuia zikusanyike.
- Param **`lr`** ni **learning rate**, ambayo huamua **ukubwa wa hatua** zinazochukuliwa wakati wa optimization process ya kusasisha parameters za model. Learning rate **ndogo** inamaanisha optimizer **hufanya updates ndogo** kwenye weights, jambo ambalo linaweza kusababisha convergence **iliyo sahihi zaidi**, lakini huenda **likapunguza kasi** ya training. Learning rate **kubwa** inaweza kuharakisha training, lakini **ina hatari ya kuvuka** minimum ya loss function (**kuruka juu ya** pointi ambayo loss function imepunguzwa zaidi).
- **Weight Decay** hubadilisha hatua ya **Loss Calculation** kwa kuongeza term ya ziada inayoadhibu weights kubwa. Hii huihimiza optimizer kutafuta solutions zenye weights ndogo, ikisawazisha kati ya ku-fit data vizuri na kuifanya model iwe rahisi, na kuzuia overfitting katika machine learning models kwa kuzuia model kuipa feature yoyote moja umuhimu mkubwa kupita kiasi.
- Traditional optimizers kama SGD iliyo na L2 regularization huunganisha weight decay na gradient ya loss function. Hata hivyo, **AdamW** (variant ya Adam optimizer) hutenganisha weight decay na gradient update, jambo linalosababisha regularization yenye ufanisi zaidi.
- Device ya kutumia kwa training
- Idadi ya epochs: Idadi ya mara za kupitia training data
- Evaluation frequency: Mara ngapi ya kuita `evaluate_model`
- Evaluation iteration: Idadi ya batches za kutumia wakati wa kutathmini hali ya sasa ya model unapoita `generate_and_print_sample`
- Start context: Sentensi ya kuanzia ya kutumia unapoita `generate_and_print_sample`
- Tokenizer
```python
# Functions to train the data
def train_model_simple(model, train_loader, val_loader, optimizer, device, num_epochs,
eval_freq, eval_iter, start_context, tokenizer):
# Initialize lists to track losses and tokens seen
train_losses, val_losses, track_tokens_seen = [], [], []
tokens_seen, global_step = 0, -1

# Main training loop
for epoch in range(num_epochs):
model.train()  # Set model to training mode

for input_batch, target_batch in train_loader:
optimizer.zero_grad() # Reset loss gradients from previous batch iteration
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss.backward() # Calculate loss gradients
optimizer.step() # Update model weights using loss gradients
tokens_seen += input_batch.numel()
global_step += 1

# Optional evaluation step
if global_step % eval_freq == 0:
train_loss, val_loss = evaluate_model(
model, train_loader, val_loader, device, eval_iter)
train_losses.append(train_loss)
val_losses.append(val_loss)
track_tokens_seen.append(tokens_seen)
print(f"Ep {epoch+1} (Step {global_step:06d}): "
f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")

# Print a sample text after each epoch
generate_and_print_sample(
model, tokenizer, device, start_context
)

return train_losses, val_losses, track_tokens_seen


def evaluate_model(model, train_loader, val_loader, device, eval_iter):
model.eval() # Set in eval mode to avoid dropout
with torch.no_grad():
train_loss = calc_loss_loader(train_loader, model, device, num_batches=eval_iter)
val_loss = calc_loss_loader(val_loader, model, device, num_batches=eval_iter)
model.train() # Back to training model applying all the configurations
return train_loss, val_loss


def generate_and_print_sample(model, tokenizer, device, start_context):
model.eval() # Set in eval mode to avoid dropout
context_size = model.pos_emb.weight.shape[0]
encoded = text_to_token_ids(start_context, tokenizer).to(device)
with torch.no_grad():
token_ids = generate_text(
model=model, idx=encoded,
max_new_tokens=50, context_size=context_size
)
decoded_text = token_ids_to_text(token_ids, tokenizer)
print(decoded_text.replace("\n", " "))  # Compact print format
model.train() # Back to training model applying all the configurations
```
> [!TIP]
> Ili kuboresha learning rate, kuna mbinu kadhaa muhimu zinazoitwa **linear warmup** na **cosine decay.**
>
> **Linear warmup** inajumuisha kuweka learning rate ya mwanzo na ya juu zaidi, kisha kuisasisha kwa mfululizo baada ya kila epoch. Hii ni kwa sababu kuanza training kwa masasisho madogo ya uzani hupunguza hatari ya model kukumbana na masasisho makubwa yasiyo thabiti wakati wa training.\
> **Cosine decay** ni mbinu ambayo **hupunguza learning rate hatua kwa hatua** kwa kufuata mkondo wa nusu-cosine **baada ya** awamu ya **warmup**, na hivyo kupunguza kasi ya masasisho ya uzani ili **kupunguza hatari ya kuvuka minima za loss** na kuhakikisha training inabaki thabiti katika awamu za baadaye.
>
> _Kumbuka kuwa maboresho haya hayajajumuishwa kwenye code iliyotangulia._

### Maboresho ya kisasa ya mafunzo kwa vitendo

Loop iliyotangulia inatosha kuelewa mechanics, lakini pre-training ya kisasa kwa kawaida huongeza optimizations kadhaa ili kuboresha throughput na stability:

- **Mixed precision (`bfloat16` / `float16`)** ili kupunguza matumizi ya memory na kuongeza kasi ya matrix multiplications. Kwa vitendo, `bfloat16` hupendelewa kwa kawaida kwenye NVIDIA datacenter GPUs za hivi karibuni kwa sababu huhifadhi exponent range pana zaidi na kwa kawaida haihitaji gradient scaling, huku `float16` mara nyingi ikitumia `GradScaler`.
- **Gradient accumulation** ili kuiga global batch size kubwa zaidi wakati batch kamili haiwezi kutoshea kwenye VRAM.
- **Gradient clipping** (kwa mfano `clip_grad_norm_`) ili kuzuia updates zinazolipuka, hasa mwanzoni mwa training.
- **Activation checkpointing** ili kubadilisha compute ya ziada kwa matumizi madogo zaidi ya memory kwa kuhesabu upya sehemu ya forward pass wakati wa backpropagation.
- **Fused / memory-efficient attention** (`scaled_dot_product_attention`) ili kupunguza gharama ya attention kwa context windows ndefu zaidi.
- **Sharded training/checkpointing** (kwa mfano FSDP / distributed checkpoints) pindi model au optimizer states zinaposhindwa tena kutoshea kwenye GPU moja.

Mfano mdogo unaochanganya baadhi ya mawazo haya katika PyTorch unaonekana hivi:
```python
accum_steps = 8
scaler = torch.amp.GradScaler("cuda")
optimizer.zero_grad(set_to_none=True)

for step, (input_batch, target_batch) in enumerate(train_loader, start=1):
with torch.autocast(device_type="cuda", dtype=torch.float16):
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss = loss / accum_steps

scaler.scale(loss).backward()

if step % accum_steps == 0:
scaler.unscale_(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
scaler.step(optimizer)
scaler.update()
optimizer.zero_grad(set_to_none=True)
```
> [!WARNING]
> Unapofanya pre-training kutoka kwa checkpoints au repositories za third-party, chukulia **loading path kama sehemu ya attack surface**. Pendelea kupakia `state_dict`s au faili za `safetensors` badala ya objects nzima za Python zilizopakiwa kwa pickle, weka `trust_remote_code=False` isipokuwa ume-audit repository, na angalia [../AI-Models-RCE.md](../AI-Models-RCE.md) kwa mifano ya RCE wakati wa kupakia model.

### Upakiaji wa checkpoints ulio salama zaidi na unaotumia memory kwa ufanisi zaidi

Kwa checkpoints kubwa, mfuatano wa kawaida wa `torch.load(...)` + `load_state_dict(...)` unaweza kuongeza memory mara mbili kwa muda (parameters zilizoanzishwa + weights zilizopakiwa). Toleo za hivi karibuni za PyTorch zimeongeza knobs muhimu sana. Kuanzia PyTorch 2.6, `weights_only=True` ndiyo default ya `torch.load()` wakati hakuna custom pickle module iliyopitishwa, lakini bado ni wazo zuri kuwa wazi kwenye mifano:

- `weights_only=True`: punguza deserialization iwe kwenye tensors / objects za msingi badala ya objects za Python zilizopakiwa kwa pickle kiholela.
- `mmap=True`: memory-map tensor storages kutoka kwenye disk badala ya kusoma kila kitu mara moja kwenye RAM.
- `with torch.device("meta"):`: tengeneza module ikiwa na tensors tupu kwanza.
- `load_state_dict(..., assign=True)`: assign tensors zilizopakiwa moja kwa moja kwenye module iliyoanzishwa kwa meta.
```python
state_dict = torch.load(
"model.pth",
map_location="cpu",
weights_only=True,
mmap=True,
)

with torch.device("meta"):
model = GPTModel(GPT_CONFIG_124M)

model.load_state_dict(state_dict, assign=True)
model.to(device)
model.eval()
```
Hii ni muhimu hasa unapopakia checkpoints zenye ukubwa wa GB nyingi kwenye workstation, kwa sababu inapunguza ongezeko la RAM wakati wa kuanzisha.
