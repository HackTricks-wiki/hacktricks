## Generisanje teksta

Da bismo trenirali model, biće potrebno da taj model može da generiše nove tokene. Zatim ćemo generisane tokene uporediti sa očekivanim kako bismo model trenirali da **nauči tokene koje treba da generiše**.

Pošto smo u prethodnim primerima već predviđali neke tokene, moguće je ponovo iskoristiti tu funkciju u ovu svrhu.

> [!TIP]
> Cilj ove šeste faze je veoma jednostavan: **Trenirati model od nule**. U tu svrhu koristiće se prethodna LLM arhitektura, uz određene petlje koje prolaze kroz skupove podataka i koriste definisane funkcije gubitka i optimizer za treniranje svih parametara modela.

## Evaluacija teksta

Da bi se trening pravilno izvršio, potrebno je proveriti predviđanja dobijena za očekivani token. Cilj treninga je maksimizovanje verovatnoće tačnog tokena, što podrazumeva povećavanje njegove verovatnoće u odnosu na druge tokene.

Da bi se maksimizovala verovatnoća tačnog tokena, težine modela moraju biti izmenjene tako da se ta verovatnoća maksimizuje. Ažuriranje težina vrši se pomoću **backpropagation**. Za to je potrebna **loss function to maximize**. U ovom slučaju, funkcija će predstavljati **razliku između izvršenog predviđanja i željenog predviđanja**.

Međutim, umesto rada sa sirovim predviđanjima, koristiće se logaritam sa bazom n. Ako je trenutno predviđanje očekivanog tokena bilo 7.4541e-05, prirodni logaritam (baza *e*) vrednosti **7.4541e-05** iznosi približno **-9.5042**.\
Zatim će, za svaki unos sa dužinom konteksta od, na primer, 5 tokena, model morati da predvidi 5 tokena, pri čemu će prva 4 tokena biti poslednji tokeni ulaza, a peti predviđeni token. Dakle, za svaki unos ćemo u tom slučaju imati 5 predviđanja (čak i ako su prva 4 bila u ulazu, model to ne zna), sa 5 očekivanih tokena, a samim tim i 5 verovatnoća koje treba maksimizovati.

Zato se, nakon izračunavanja prirodnog logaritma za svako predviđanje, računa **prosek**, uklanja se **minus** (to se naziva _cross entropy loss_), a to je **broj koji treba smanjiti što je moguće bliže nuli**, jer je prirodni logaritam broja 1 jednak 0:

```python
import tiktoken
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader


class GPTDatasetV1(Dataset):
def __init__(self, txt, tokenizer, max_length, stride):
self.input_ids = []
self.target_ids = []

# Tokenize the entire text
token_ids = tokenizer.encode(txt, allowed_special={"<|endoftext|>"})

# Use a sliding window to chunk the book into overlapping sequences of max_length
for i in range(0, len(token_ids) - max_length, stride):
input_chunk = token_ids[i:i + max_length]
target_chunk = token_ids[i + 1: i + max_length + 1]
self.input_ids.append(torch.tensor(input_chunk))
self.target_ids.append(torch.tensor(target_chunk))

def __len__(self):
return len(self.input_ids)

def __getitem__(self, idx):
return self.input_ids[idx], self.target_ids[idx]


def create_dataloader_v1(txt, batch_size=4, max_length=256,
stride=128, shuffle=True, drop_last=True, num_workers=0):
# Initialize the tokenizer
tokenizer = tiktoken.get_encoding("gpt2")

# Create dataset
dataset = GPTDatasetV1(txt, tokenizer, max_length, stride)

# Create dataloader
dataloader = DataLoader(
dataset, batch_size=batch_size, shuffle=shuffle, drop_last=drop_last, num_workers=num_workers)

return dataloader


class MultiHeadAttention(nn.Module):
def __init__(self, d_in, d_out, context_length, dropout, num_heads, qkv_bias=False):
super().__init__()
assert d_out % num_heads == 0, "d_out must be divisible by n_heads"

self.d_out = d_out
self.num_heads = num_heads
self.head_dim = d_out // num_heads  # Reduce the projection dim to match desired output dim

self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_key = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
self.out_proj = nn.Linear(d_out, d_out)  # Linear layer to combine head outputs
self.dropout = nn.Dropout(dropout)
self.register_buffer('mask', torch.triu(torch.ones(context_length, context_length), diagonal=1))

def forward(self, x):
b, num_tokens, d_in = x.shape

keys = self.W_key(x)  # Shape: (b, num_tokens, d_out)
queries = self.W_query(x)
values = self.W_value(x)

# We implicitly split the matrix by adding a `num_heads` dimension
# Unroll last dim: (b, num_tokens, d_out) -> (b, num_tokens, num_heads, head_dim)
keys = keys.view(b, num_tokens, self.num_heads, self.head_dim)
values = values.view(b, num_tokens, self.num_heads, self.head_dim)
queries = queries.view(b, num_tokens, self.num_heads, self.head_dim)

# Transpose: (b, num_tokens, num_heads, head_dim) -> (b, num_heads, num_tokens, head_dim)
keys = keys.transpose(1, 2)
queries = queries.transpose(1, 2)
values = values.transpose(1, 2)

# Compute scaled dot-product attention (aka self-attention) with a causal mask
attn_scores = queries @ keys.transpose(2, 3)  # Dot product for each head

# Original mask truncated to the number of tokens and converted to boolean
mask_bool = self.mask.bool()[:num_tokens, :num_tokens]

# Use the mask to fill attention scores
attn_scores.masked_fill_(mask_bool, -torch.inf)

attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
attn_weights = self.dropout(attn_weights)

# Shape: (b, num_tokens, num_heads, head_dim)
context_vec = (attn_weights @ values).transpose(1, 2)

# Combine heads, where self.d_out = self.num_heads * self.head_dim
context_vec = context_vec.reshape(b, num_tokens, self.d_out)
context_vec = self.out_proj(context_vec)  # optional projection

return context_vec


class LayerNorm(nn.Module):
def __init__(self, emb_dim):
super().__init__()
self.eps = 1e-5
self.scale = nn.Parameter(torch.ones(emb_dim))
self.shift = nn.Parameter(torch.zeros(emb_dim))

def forward(self, x):
mean = x.mean(dim=-1, keepdim=True)
var = x.var(dim=-1, keepdim=True, unbiased=False)
norm_x = (x - mean) / torch.sqrt(var + self.eps)
return self.scale * norm_x + self.shift


class GELU(nn.Module):
def __init__(self):
super().__init__()

def forward(self, x):
return 0.5 * x * (1 + torch.tanh(
torch.sqrt(torch.tensor(2.0 / torch.pi)) *
(x + 0.044715 * torch.pow(x, 3))
))


class FeedForward(nn.Module):
def __init__(self, cfg):
super().__init__()
self.layers = nn.Sequential(
nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),
GELU(),
nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"]),
)

def forward(self, x):
return self.layers(x)


class TransformerBlock(nn.Module):
def __init__(self, cfg):
super().__init__()
self.att = MultiHeadAttention(
d_in=cfg["emb_dim"],
d_out=cfg["emb_dim"],
context_length=cfg["context_length"],
num_heads=cfg["n_heads"],
dropout=cfg["drop_rate"],
qkv_bias=cfg["qkv_bias"])
self.ff = FeedForward(cfg)
self.norm1 = LayerNorm(cfg["emb_dim"])
self.norm2 = LayerNorm(cfg["emb_dim"])
self.drop_shortcut = nn.Dropout(cfg["drop_rate"])

def forward(self, x):
# Shortcut connection for attention block
shortcut = x
x = self.norm1(x)
x = self.att(x)   # Shape [batch_size, num_tokens, emb_size]
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

# Shortcut connection for feed-forward block
shortcut = x
x = self.norm2(x)
x = self.ff(x)
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

return x


class GPTModel(nn.Module):
def __init__(self, cfg):
super().__init__()
self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])
self.drop_emb = nn.Dropout(cfg["drop_rate"])

self.trf_blocks = nn.Sequential(
*[TransformerBlock(cfg) for _ in range(cfg["n_layers"])])

self.final_norm = LayerNorm(cfg["emb_dim"])
self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

def forward(self, in_idx):
batch_size, seq_len = in_idx.shape
tok_embeds = self.tok_emb(in_idx)
pos_embeds = self.pos_emb(torch.arange(seq_len, device=in_idx.device))
x = tok_embeds + pos_embeds  # Shape [batch_size, num_tokens, emb_size]
x = self.drop_emb(x)
x = self.trf_blocks(x)
x = self.final_norm(x)
logits = self.out_head(x)
return logits
```

### Funkcije za generisanje teksta

U prethodnom odeljku funkcija je jednostavno uzimala **najverovatniji token** nakon dobijanja vrednosti `logits`. Međutim, to znači da će se za svaki unos uvek generisati isti izlaz, što je čini veoma determinističkom.

Sledeća funkcija `generate_text` primenjuje koncepte `top-k`, `temperature` i `multinomial`.

- **`top-k`** znači da ćemo početi da postavljamo na `-inf` verovatnoće svih tokena, osim za k najboljih tokena. Dakle, ako je k=3, pre donošenja odluke samo 3 najverovatnija tokena imaće verovatnoću različitu od `-inf`.
- **`temperature`** znači da će svaka verovatnoća biti podeljena vrednošću temperature. Vrednost `0.1` će povećati razliku između najveće i najmanje verovatnoće, dok će je temperatura od `5`, na primer, učiniti ravnomernijom. Ovo pomaže u povećanju varijacija u odgovorima koje želimo da LLM generiše.
- Nakon primene temperature, funkcija **`softmax`** se ponovo primenjuje kako bi zbir verovatnoća svih preostalih tokena bio 1.
- Na kraju, umesto izbora tokena sa najvećom verovatnoćom, funkcija **`multinomial`** se primenjuje za **predviđanje sledećeg tokena prema konačnim verovatnoćama**. Dakle, ako token 1 ima verovatnoću od 70%, token 2 od 20%, a token 3 od 10%, token 1 će biti izabran u 70% slučajeva, token 2 u 20% slučajeva, a token 3 u 10% slučajeva.

```python
# Generate text function
def generate_text(model, idx, max_new_tokens, context_size, temperature=0.0, top_k=None, eos_id=None):

# For-loop is the same as before: Get logits, and only focus on last time step
for _ in range(max_new_tokens):
idx_cond = idx[:, -context_size:]
with torch.no_grad():
logits = model(idx_cond)
logits = logits[:, -1, :]

# New: Filter logits with top_k sampling
if top_k is not None:
# Keep only top_k values
top_logits, _ = torch.topk(logits, top_k)
min_val = top_logits[:, -1]
logits = torch.where(logits < min_val, torch.tensor(float("-inf")).to(logits.device), logits)

# New: Apply temperature scaling
if temperature > 0.0:
logits = logits / temperature

# Apply softmax to get probabilities
probs = torch.softmax(logits, dim=-1)  # (batch_size, context_len)

# Sample from the distribution
idx_next = torch.multinomial(probs, num_samples=1)  # (batch_size, 1)

# Otherwise same as before: get idx of the vocab entry with the highest logits value
else:
idx_next = torch.argmax(logits, dim=-1, keepdim=True)  # (batch_size, 1)

if idx_next == eos_id:  # Stop generating early if end-of-sequence token is encountered and eos_id is specified
break

# Same as before: append sampled index to the running sequence
idx = torch.cat((idx, idx_next), dim=1)  # (batch_size, num_tokens+1)

return idx
```
> [!TIP]
> Postoji uobičajena alternativa za `top-k` koja se zove [**`top-p`**](https://en.wikipedia.org/wiki/Top-p_sampling), poznata i kao nucleus sampling, koja umesto uzimanja k uzoraka sa najvećom verovatnoćom, **organizuje** ceo rezultujući **vocabulary** prema verovatnoćama i **sabira** ih od najveće verovatnoće ka najmanjoj, sve dok se ne dostigne **prag**.
>
> Zatim će se uzeti u obzir **samo te reči** iz vocabulary-ja, u skladu sa njihovim relativnim verovatnoćama.
>
> Ovo omogućava da nije potrebno izabrati broj `k` uzoraka, jer optimalna vrednost k može biti različita u svakom slučaju, već je dovoljan **samo prag**.
>
> _Imajte na umu da ovo poboljšanje nije uključeno u prethodni kod._

> [!TIP]
> Drugi način za poboljšanje generisanog teksta jeste korišćenje **Beam search** umesto greedy search-a korišćenog u ovom primeru.\
> Za razliku od greedy search-a, koji u svakom koraku bira sledeću reč sa najvećom verovatnoćom i gradi jednu sekvencu, **beam search prati 𝑘 k najbolje rangiranih delimičnih sekvenci** (zvanih „beams“) u svakom koraku. Istovremenim istraživanjem više mogućnosti, on predstavlja ravnotežu između efikasnosti i kvaliteta, povećavajući šanse za **pronalaženje bolje ukupne** sekvence koju bi greedy pristup mogao da propusti zbog ranih, suboptimalnih izbora.
>
> _Imajte na umu da ovo poboljšanje nije uključeno u prethodni kod._

### Funkcije gubitka

Funkcija **`calc_loss_batch`** izračunava cross entropy predikcije jednog batch-a.\
Funkcija **`calc_loss_loader`** dobija cross entropy svih batch-eva i izračunava **prosečnu cross entropy**.
```python
# Define loss functions
def calc_loss_batch(input_batch, target_batch, model, device):
input_batch, target_batch = input_batch.to(device), target_batch.to(device)
logits = model(input_batch)
loss = torch.nn.functional.cross_entropy(logits.flatten(0, 1), target_batch.flatten())
return loss

def calc_loss_loader(data_loader, model, device, num_batches=None):
total_loss = 0.
if len(data_loader) == 0:
return float("nan")
elif num_batches is None:
num_batches = len(data_loader)
else:
# Reduce the number of batches to match the total number of batches in the data loader
# if num_batches exceeds the number of batches in the data loader
num_batches = min(num_batches, len(data_loader))
for i, (input_batch, target_batch) in enumerate(data_loader):
if i < num_batches:
loss = calc_loss_batch(input_batch, target_batch, model, device)
total_loss += loss.item()
else:
break
return total_loss / num_batches
```
> [!TIP]
> **Gradient clipping** je tehnika koja se koristi za poboljšanje **stabilnosti treniranja** velikih neuronskih mreža postavljanjem **maksimalnog praga** za veličine gradijenata. Kada gradijenti premaše unapred definisani `max_norm`, proporcionalno se smanjuju kako bi se osiguralo da ažuriranja parametara modela ostanu u prihvatljivom opsegu, čime se sprečavaju problemi poput eksplodirajućih gradijenata i obezbeđuje kontrolisanije i stabilnije treniranje.
>
> _Imajte na umu da ovo poboljšanje nije uključeno u prethodni kod._

### Funkcije za trening

Funkcija `generate_and_print_sample` samo će dobiti kontekst i generisati nekoliko tokena kako bi se stekao utisak o tome koliko je model dobar u tom trenutku. Ovo poziva `train_model_simple` pri svakom koraku.

Funkcija `evaluate_model` poziva se onoliko često koliko je navedeno u funkciji za trening i koristi se za merenje gubitka pri treniranju i gubitka pri validaciji u datoj tački treniranja modela.

Zatim je tu velika funkcija `train_model_simple`, koja zapravo trenira model. Očekuje:

- Data loader za trening (sa podacima koji su već razdvojeni i pripremljeni za trening)
- Validator loader
- **optimizer** koji će se koristiti tokom treninga: Ovo je funkcija koja će koristiti gradijente i ažurirati parametre kako bi smanjila gubitak. U ovom slučaju, kao što ćete videti, koristi se `AdamW`, ali postoji još mnogo drugih.
- `optimizer.zero_grad()` poziva se u svakoj iteraciji kako bi se resetovali gradijenti i sprečilo njihovo akumuliranje.
- Parametar **`lr`** predstavlja **learning rate**, koji određuje **veličinu koraka** tokom procesa optimizacije pri ažuriranju parametara modela. **Manji** learning rate znači da optimizer vrši **manja ažuriranja** težina, što može dovesti do **preciznije** konvergencije, ali može i **usporiti** trening. **Veći** learning rate može ubrzati trening, ali postoji **rizik od preskakanja** minimuma funkcije gubitka (**preskakanja** tačke u kojoj je funkcija gubitka minimalna).
- **Weight Decay** menja korak **izračunavanja gubitka** dodavanjem dodatnog člana koji kažnjava velike težine. Ovo podstiče optimizer da pronađe rešenja sa manjim težinama, praveći ravnotežu između dobrog uklapanja podataka i održavanja jednostavnosti modela, čime se sprečava overfitting u machine learning modelima tako što se model odvraća od pridavanja prevelike važnosti bilo kojoj pojedinačnoj osobini.
- Tradicionalni optimizatori, kao što je SGD sa L2 regularizacijom, povezuju weight decay sa gradijentom funkcije gubitka. Međutim, **AdamW** (varijanta Adam optimizatora) odvaja weight decay od ažuriranja gradijenta, što dovodi do efikasnije regularizacije.
- Uređaj koji će se koristiti za trening
- Broj epoha: Broj prolazaka kroz podatke za trening
- Učestalost evaluacije: Učestalost pozivanja funkcije `evaluate_model`
- Iteracija evaluacije: Broj batch-eva koji će se koristiti pri evaluaciji trenutnog stanja modela prilikom pozivanja funkcije `generate_and_print_sample`
- Početni kontekst: Početna rečenica koja će se koristiti prilikom pozivanja funkcije `generate_and_print_sample`
- Tokenizer
```python
# Functions to train the data
def train_model_simple(model, train_loader, val_loader, optimizer, device, num_epochs,
eval_freq, eval_iter, start_context, tokenizer):
# Initialize lists to track losses and tokens seen
train_losses, val_losses, track_tokens_seen = [], [], []
tokens_seen, global_step = 0, -1

# Main training loop
for epoch in range(num_epochs):
model.train()  # Set model to training mode

for input_batch, target_batch in train_loader:
optimizer.zero_grad() # Reset loss gradients from previous batch iteration
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss.backward() # Calculate loss gradients
optimizer.step() # Update model weights using loss gradients
tokens_seen += input_batch.numel()
global_step += 1

# Optional evaluation step
if global_step % eval_freq == 0:
train_loss, val_loss = evaluate_model(
model, train_loader, val_loader, device, eval_iter)
train_losses.append(train_loss)
val_losses.append(val_loss)
track_tokens_seen.append(tokens_seen)
print(f"Ep {epoch+1} (Step {global_step:06d}): "
f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")

# Print a sample text after each epoch
generate_and_print_sample(
model, tokenizer, device, start_context
)

return train_losses, val_losses, track_tokens_seen


def evaluate_model(model, train_loader, val_loader, device, eval_iter):
model.eval() # Set in eval mode to avoid dropout
with torch.no_grad():
train_loss = calc_loss_loader(train_loader, model, device, num_batches=eval_iter)
val_loss = calc_loss_loader(val_loader, model, device, num_batches=eval_iter)
model.train() # Back to training model applying all the configurations
return train_loss, val_loss


def generate_and_print_sample(model, tokenizer, device, start_context):
model.eval() # Set in eval mode to avoid dropout
context_size = model.pos_emb.weight.shape[0]
encoded = text_to_token_ids(start_context, tokenizer).to(device)
with torch.no_grad():
token_ids = generate_text(
model=model, idx=encoded,
max_new_tokens=50, context_size=context_size
)
decoded_text = token_ids_to_text(token_ids, tokenizer)
print(decoded_text.replace("\n", " "))  # Compact print format
model.train() # Back to training model applying all the configurations
```
> [!TIP]
> Za poboljšanje stope učenja postoje dve relevantne tehnike koje se nazivaju **linear warmup** i **cosine decay.**
>
> **Linear warmup** se sastoji od definisanja početne i maksimalne stope učenja i njenog doslednog ažuriranja nakon svake epohe. Ovo je zato što započinjanje obuke manjim ažuriranjima težina smanjuje rizik da model tokom faze obuke naiđe na velika, destabilizujuća ažuriranja.\
> **Cosine decay** je tehnika koja **postepeno smanjuje stopu učenja** prateći polukosinusnu krivu **nakon faze warmup**, usporavajući ažuriranja težina kako bi se **smanjio rizik od prekoračenja** minimuma funkcije gubitka i obezbedila stabilnost obuke u kasnijim fazama.
>
> _Imajte na umu da ova poboljšanja nisu uključena u prethodni kod._

### Praktična poboljšanja savremenog treninga

Prethodna petlja je dovoljna za razumevanje mehanike, ali savremeni pre-training obično dodaje nekoliko optimizacija radi poboljšanja propusnosti i stabilnosti:

- **Mixed precision (`bfloat16` / `float16`)** za smanjenje potrošnje memorije i ubrzavanje množenja matrica. U praksi se `bfloat16` obično preferira na novijim NVIDIA datacenter GPU-ovima jer zadržava širi opseg eksponenta i obično ne zahteva skaliranje gradijenata, dok `float16` često koristi `GradScaler`.
- **Gradient accumulation** za simulaciju veće globalne veličine batch-a kada ceo batch ne može da stane u VRAM.
- **Gradient clipping** (na primer `clip_grad_norm_`) za sprečavanje eksplozivnih ažuriranja, naročito na početku treninga.
- **Activation checkpointing** za zamenu dodatnog računanja za znatno manju potrošnju memorije, ponovnim izračunavanjem dela forward pass-a tokom backpropagation-a.
- **Fused / memory-efficient attention** (`scaled_dot_product_attention`) za smanjenje troškova attention-a kod dužih context window-a.
- **Sharded training/checkpointing** (na primer FSDP / distributed checkpoints) kada model ili stanja optimizatora više ne mogu da stanu na jedan GPU.

Minimalni primer u PyTorch-u koji kombinuje neke od ovih ideja izgleda ovako:
```python
accum_steps = 8
scaler = torch.amp.GradScaler("cuda")
optimizer.zero_grad(set_to_none=True)

for step, (input_batch, target_batch) in enumerate(train_loader, start=1):
with torch.autocast(device_type="cuda", dtype=torch.float16):
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss = loss / accum_steps

scaler.scale(loss).backward()

if step % accum_steps == 0:
scaler.unscale_(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
scaler.step(optimizer)
scaler.update()
optimizer.zero_grad(set_to_none=True)
```
> [!WARNING]
> Prilikom pre-traininga iz checkpoint-a ili third-party repozitorijuma, tretirajte **putanju učitavanja kao deo attack surface-a**. Dajte prednost učitavanju `state_dict` ili `safetensors` fajlova umesto celih pickle-ovanih Python objekata, zadržite `trust_remote_code=False` osim ako niste izvršili audit repozitorijuma i pogledajte [../AI-Models-RCE.md](../AI-Models-RCE.md) za primere RCE-a pri učitavanju modela.

### Bezbednije i memorijski efikasnije učitavanje checkpoint-a

Kod velikih checkpoint-a, naivni redosled `torch.load(...)` + `load_state_dict(...)` može privremeno udvostručiti memoriju (inicijalizovani parametri + učitane težine). Novije verzije PyTorch-a dodale su nekoliko veoma korisnih opcija. Počevši od PyTorch 2.6, `weights_only=True` je podrazumevana vrednost za `torch.load()` kada nije prosleđen prilagođeni pickle modul, ali je i dalje dobra praksa da to eksplicitno navedete u primerima:

- `weights_only=True`: ograničava deserijalizaciju na tenzore / primitivne objekte umesto na proizvoljne pickle-ovane Python objekte.
- `mmap=True`: memorijski mapira skladišta tenzora sa diska umesto da sve unapred učita u RAM.
- `with torch.device("meta"):`: najpre kreira modul sa praznim tenzorima.
- `load_state_dict(..., assign=True)`: direktno dodeljuje učitane tenzore modulu inicijalizovanom pomoću `meta`.
```python
state_dict = torch.load(
"model.pth",
map_location="cpu",
weights_only=True,
mmap=True,
)

with torch.device("meta"):
model = GPTModel(GPT_CONFIG_124M)

model.load_state_dict(state_dict, assign=True)
model.to(device)
model.eval()
```
Ovo je posebno korisno kada se checkpoint-i veličine više GB učitavaju na workstation, jer smanjuje skokove u RAM-u tokom pokretanja.

```python
torch_dtype=torch.bfloat16,
device_map="auto",
low_cpu_mem_usage=True,
trust_remote_code=False,
)
```
`device_map="auto"` je veoma praktičan kada se ceo checkpoint ne uklapa u jedan accelerator, jer loader može da rasporedi layers između GPU / CPU memorije. Za velike repozitorijume, `save_pretrained(..., max_shard_size="5GB")` ili odgovarajući Hub checkpoints generišu **sharded checkpoints** i indeks, što omogućava njihovo inkrementalno učitavanje umesto zahteva za jednom ogromnom datotekom u RAM-u.

Ako treba da pregledate ili izdvojite tensors bez učitavanja cele datoteke, `safetensors` takođe podržava delimična čitanja:
```python
from safetensors import safe_open

with safe_open("model.safetensors", framework="pt", device="cpu") as f:
emb_slice = f.get_slice("model.embed_tokens.weight")
vocab_size, hidden_dim = emb_slice.get_shape()
first_half = emb_slice[:, : hidden_dim // 2]
```
