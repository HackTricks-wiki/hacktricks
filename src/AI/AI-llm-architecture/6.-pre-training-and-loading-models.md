## Text Generation

모델을 train하려면 해당 모델이 새로운 토큰을 생성할 수 있어야 합니다. 그런 다음 생성된 토큰을 예상 토큰과 비교하여 **모델이 생성해야 하는 토큰을 학습하도록** train합니다.

이전 예제에서 이미 일부 토큰을 예측했으므로, 이 목적을 위해 해당 함수를 재사용할 수 있습니다.

> [!TIP]
> 이 여섯 번째 단계의 목표는 매우 간단합니다: **모델을 처음부터 Train하기**. 이를 위해 이전 LLM architecture를 사용하고, 정의된 loss functions와 optimizer를 사용해 데이터 세트를 순회하는 일부 loop를 통해 모델의 모든 parameter를 train합니다.

## Text Evaluation

올바르게 train하려면 예상 토큰에 대해 얻은 prediction을 측정하고 확인해야 합니다. train의 목표는 올바른 토큰의 likelihood를 최대화하는 것이며, 이는 다른 토큰에 비해 해당 토큰의 probability를 높이는 것을 의미합니다.

올바른 토큰의 probability를 최대화하려면 해당 probability가 최대화되도록 모델의 weights를 수정해야 합니다. weights의 update는 **backpropagation**을 통해 수행됩니다. 이를 위해서는 **maximize할 loss function**이 필요합니다. 이 경우 이 function은 **수행된 prediction과 원하는 prediction의 차이**입니다.

그러나 raw prediction을 직접 사용하는 대신, base n인 logarithm을 사용합니다. 따라서 예상 토큰에 대한 현재 prediction이 7.4541e-05라면, **7.4541e-05**의 natural logarithm (base *e*)은 약 **-9.5042**입니다.\
그런 다음 예를 들어 context length가 5개 토큰인 각 entry에 대해, 모델은 5개의 토큰을 predict해야 합니다. 이때 처음 4개 토큰은 input의 마지막 토큰이고 다섯 번째 토큰은 predicted token입니다. 따라서 각 entry에는 이 경우 5개의 prediction이 존재합니다 (처음 4개가 input에 포함되어 있더라도 모델은 이를 알지 못합니다). 또한 예상 토큰 5개가 있으므로 maximize할 probability도 5개가 존재합니다.

따라서 각 prediction에 natural logarithm을 적용한 후 **average**를 계산하고, **minus symbol을 제거합니다** (이를 _cross entropy loss_라고 합니다). 그리고 natural logarithm of 1이 0이므로 **가능한 한 0에 가깝게 줄여야 하는 값**이 됩니다:

모델이 얼마나 잘 작동하는지 측정하는 또 다른 방법은 perplexity라고 합니다. **Perplexity**는 probability model이 sample을 얼마나 잘 predict하는지 평가하는 데 사용되는 metric입니다. language modelling에서 이는 sequence의 다음 토큰을 predict할 때 **모델의 uncertainty**를 나타냅니다.\
예를 들어 perplexity 값이 48725라는 것은 토큰을 predict해야 할 때 vocabulary에 있는 48,725개의 토큰 중 어떤 것이 올바른 토큰인지 확신하지 못한다는 의미입니다.

import tiktoken
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader


class GPTDatasetV1(Dataset):
def __init__(self, txt, tokenizer, max_length, stride):
self.input_ids = []
self.target_ids = []

# Tokenize the entire text
token_ids = tokenizer.encode(txt, allowed_special={"<|endoftext|>"})

# Use a sliding window to chunk the book into overlapping sequences of max_length
for i in range(0, len(token_ids) - max_length, stride):
input_chunk = token_ids[i:i + max_length]
target_chunk = token_ids[i + 1: i + max_length + 1]
self.input_ids.append(torch.tensor(input_chunk))
self.target_ids.append(torch.tensor(target_chunk))

def __len__(self):
return len(self.input_ids)

def __getitem__(self, idx):
return self.input_ids[idx], self.target_ids[idx]


def create_dataloader_v1(txt, batch_size=4, max_length=256,
stride=128, shuffle=True, drop_last=True, num_workers=0):
# Initialize the tokenizer
tokenizer = tiktoken.get_encoding("gpt2")

# Create dataset
dataset = GPTDatasetV1(txt, tokenizer, max_length, stride)

# Create dataloader
dataloader = DataLoader(
dataset, batch_size=batch_size, shuffle=shuffle, drop_last=drop_last, num_workers=num_workers)

return dataloader


class MultiHeadAttention(nn.Module):
def __init__(self, d_in, d_out, context_length, dropout, num_heads, qkv_bias=False):
super().__init__()
assert d_out % num_heads == 0, "d_out must be divisible by n_heads"

self.d_out = d_out
self.num_heads = num_heads
self.head_dim = d_out // num_heads  # Reduce the projection dim to match desired output dim

self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_key = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
self.out_proj = nn.Linear(d_out, d_out)  # Linear layer to combine head outputs
self.dropout = nn.Dropout(dropout)
self.register_buffer('mask', torch.triu(torch.ones(context_length, context_length), diagonal=1))

def forward(self, x):
b, num_tokens, d_in = x.shape

keys = self.W_key(x)  # Shape: (b, num_tokens, d_out)
queries = self.W_query(x)
values = self.W_value(x)

# We implicitly split the matrix by adding a `num_heads` dimension
# Unroll last dim: (b, num_tokens, d_out) -> (b, num_tokens, num_heads, head_dim)
keys = keys.view(b, num_tokens, self.num_heads, self.head_dim)
values = values.view(b, num_tokens, self.num_heads, self.head_dim)
queries = queries.view(b, num_tokens, self.num_heads, self.head_dim)

# Transpose: (b, num_tokens, num_heads, head_dim) -> (b, num_heads, num_tokens, head_dim)
keys = keys.transpose(1, 2)
queries = queries.transpose(1, 2)
values = values.transpose(1, 2)

# Compute scaled dot-product attention (aka self-attention) with a causal mask
attn_scores = queries @ keys.transpose(2, 3)  # Dot product for each head

# Original mask truncated to the number of tokens and converted to boolean
mask_bool = self.mask.bool()[:num_tokens, :num_tokens]

# Use the mask to fill attention scores
attn_scores.masked_fill_(mask_bool, -torch.inf)

attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
attn_weights = self.dropout(attn_weights)

# Shape: (b, num_tokens, num_heads, head_dim)
context_vec = (attn_weights @ values).transpose(1, 2)

# Combine heads, where self.d_out = self.num_heads * self.head_dim
context_vec = context_vec.reshape(b, num_tokens, self.d_out)
context_vec = self.out_proj(context_vec)  # optional projection

return context_vec


class LayerNorm(nn.Module):
def __init__(self, emb_dim):
super().__init__()
self.eps = 1e-5
self.scale = nn.Parameter(torch.ones(emb_dim))
self.shift = nn.Parameter(torch.zeros(emb_dim))

def forward(self, x):
mean = x.mean(dim=-1, keepdim=True)
var = x.var(dim=-1, keepdim=True, unbiased=False)
norm_x = (x - mean) / torch.sqrt(var + self.eps)
return self.scale * norm_x + self.shift


class GELU(nn.Module):
def __init__(self):
super().__init__()

def forward(self, x):
return 0.5 * x * (1 + torch.tanh(
torch.sqrt(torch.tensor(2.0 / torch.pi)) *
(x + 0.044715 * torch.pow(x, 3))
))


class FeedForward(nn.Module):
def __init__(self, cfg):
super().__init__()
self.layers = nn.Sequential(
nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),
GELU(),
nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"]),
)

def forward(self, x):
return self.layers(x)


class TransformerBlock(nn.Module):
def __init__(self, cfg):
super().__init__()
self.att = MultiHeadAttention(
d_in=cfg["emb_dim"],
d_out=cfg["emb_dim"],
context_length=cfg["context_length"],
num_heads=cfg["n_heads"],
dropout=cfg["drop_rate"],
qkv_bias=cfg["qkv_bias"])
self.ff = FeedForward(cfg)
self.norm1 = LayerNorm(cfg["emb_dim"])
self.norm2 = LayerNorm(cfg["emb_dim"])
self.drop_shortcut = nn.Dropout(cfg["drop_rate"])

def forward(self, x):
# Shortcut connection for attention block
shortcut = x
x = self.norm1(x)
x = self.att(x)   # Shape [batch_size, num_tokens, emb_size]
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

# Shortcut connection for feed-forward block
shortcut = x
x = self.norm2(x)
x = self.ff(x)
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

return x


class GPTModel(nn.Module):
def __init__(self, cfg):
super().__init__()
self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])
self.drop_emb = nn.Dropout(cfg["drop_rate"])

self.trf_blocks = nn.Sequential(
*[TransformerBlock(cfg) for _ in range(cfg["n_layers"])])

self.final_norm = LayerNorm(cfg["emb_dim"])
self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

def forward(self, in_idx):
batch_size, seq_len = in_idx.shape
tok_embeds = self.tok_emb(in_idx)
pos_embeds = self.pos_emb(torch.arange(seq_len, device=in_idx.device))
x = tok_embeds + pos_embeds  # Shape [batch_size, num_tokens, emb_size]
x = self.drop_emb(x)
x = self.trf_blocks(x)
x = self.final_norm(x)
logits = self.out_head(x)
return logits

logits = model(input_batch)
loss = torch.nn.functional.cross_entropy(logits.flatten(0, 1), target_batch.flatten())

def calc_loss_loader(data_loader, model, device, num_batches=None):
total_loss = 0.
if len(data_loader) == 0:
return float("nan")
elif num_batches is None:
num_batches = len(data_loader)
else:
# Reduce the number of batches to match the total number of batches in the data loader

eval_freq, eval_iter, start_context, tokenizer):

### 텍스트 생성 함수

이전 섹션에서는 logits를 얻은 후 **가장 확률이 높은 token**만 선택하는 함수를 살펴보았습니다. 그러나 이렇게 하면 각 입력에 대해 항상 동일한 출력이 생성되므로 매우 결정적입니다.

다음 `generate_text` 함수는 `top-k`, `temperature` 및 `multinomial` 개념을 적용합니다.

- **`top-k`**는 상위 k개의 token을 제외한 모든 token의 확률을 `-inf`로 줄이는 것을 의미합니다. 따라서 k=3인 경우, 선택을 수행하기 전에 가장 확률이 높은 3개의 token만 `-inf`와 다른 확률을 갖게 됩니다.
- **`temperature`**는 모든 확률을 temperature 값으로 나누는 것을 의미합니다. `0.1` 값은 가장 높은 확률과 가장 낮은 확률의 차이를 키우며, 반면 temperature가 예를 들어 `5`이면 확률 분포가 더 평평해집니다. 이는 LLM의 응답에 원하는 변화를 높이는 데 도움이 됩니다.
- temperature를 적용한 후 **`softmax`** 함수가 다시 적용되어 남아 있는 모든 token의 전체 확률 합이 1이 되도록 합니다.
- 마지막으로 확률이 가장 높은 token을 선택하는 대신, **최종 확률에 따라 다음 token을 예측하기 위해** `multinomial` 함수가 적용됩니다. 예를 들어 token 1의 확률이 70%, token 2가 20%, token 3이 10%라면, 70%의 경우 token 1이 선택되고 20%의 경우 token 2가 선택되며 10%의 경우 token 3이 선택됩니다.
```python
# Generate text function
def generate_text(model, idx, max_new_tokens, context_size, temperature=0.0, top_k=None, eos_id=None):

# For-loop is the same as before: Get logits, and only focus on last time step
for _ in range(max_new_tokens):
idx_cond = idx[:, -context_size:]
with torch.no_grad():
logits = model(idx_cond)
logits = logits[:, -1, :]

# New: Filter logits with top_k sampling
if top_k is not None:
# Keep only top_k values
top_logits, _ = torch.topk(logits, top_k)
min_val = top_logits[:, -1]
logits = torch.where(logits < min_val, torch.tensor(float("-inf")).to(logits.device), logits)

# New: Apply temperature scaling
if temperature > 0.0:
logits = logits / temperature

# Apply softmax to get probabilities
probs = torch.softmax(logits, dim=-1)  # (batch_size, context_len)

# Sample from the distribution
idx_next = torch.multinomial(probs, num_samples=1)  # (batch_size, 1)

# Otherwise same as before: get idx of the vocab entry with the highest logits value
else:
idx_next = torch.argmax(logits, dim=-1, keepdim=True)  # (batch_size, 1)

if idx_next == eos_id:  # Stop generating early if end-of-sequence token is encountered and eos_id is specified
break

# Same as before: append sampled index to the running sequence
idx = torch.cat((idx, idx_next), dim=1)  # (batch_size, num_tokens+1)

return idx
```
> [!TIP]
> `top-k`의 일반적인 대안으로 [**`top-p`**](https://en.wikipedia.org/wiki/Top-p_sampling)가 있으며, nucleus sampling이라고도 합니다. 이는 확률이 가장 높은 k개의 샘플을 가져오는 대신, 결과로 나온 모든 **vocabulary**를 확률에 따라 **정렬**하고, 가장 높은 확률부터 가장 낮은 확률 순서로 **threshold에 도달할 때까지 합산**합니다.
>
> 그런 다음 **vocabulary의 해당 단어들만** 상대적 확률에 따라 고려됩니다.
>
> 이렇게 하면 k개의 샘플 수를 선택할 필요 없이 **threshold만** 선택하면 됩니다. 최적의 k는 각 경우에 따라 달라질 수 있기 때문입니다.
>
> _이 개선 사항은 앞의 코드에는 포함되어 있지 않습니다._

> [!TIP]
> 생성된 텍스트를 개선하는 또 다른 방법은 이 예제에서 사용한 greedy search 대신 **Beam search**를 사용하는 것입니다.\
> 각 단계에서 가장 확률이 높은 다음 단어를 선택하고 하나의 시퀀스를 생성하는 greedy search와 달리, **beam search는 각 단계에서 점수가 가장 높은 상위 𝑘 k개의 부분 시퀀스**(이를 "beams"라고 함)를 추적합니다. 여러 가능성을 동시에 탐색함으로써 효율성과 품질 사이의 균형을 유지하며, 초기의 최적이 아닌 선택으로 인해 greedy 방식이 놓칠 수 있는 **더 나은 전체 시퀀스를 찾을** 가능성을 높입니다.
>
> _이 개선 사항은 앞의 코드에는 포함되어 있지 않습니다._

### 손실 함수

**`calc_loss_batch`** 함수는 단일 batch의 예측에 대한 cross entropy를 계산합니다.\
**`calc_loss_loader`**는 모든 batch의 cross entropy를 구한 뒤 **평균 cross entropy**를 계산합니다.
```python
# Define loss functions
def calc_loss_batch(input_batch, target_batch, model, device):
input_batch, target_batch = input_batch.to(device), target_batch.to(device)
logits = model(input_batch)
loss = torch.nn.functional.cross_entropy(logits.flatten(0, 1), target_batch.flatten())
return loss

def calc_loss_loader(data_loader, model, device, num_batches=None):
total_loss = 0.
if len(data_loader) == 0:
return float("nan")
elif num_batches is None:
num_batches = len(data_loader)
else:
# Reduce the number of batches to match the total number of batches in the data loader
# if num_batches exceeds the number of batches in the data loader
num_batches = min(num_batches, len(data_loader))
for i, (input_batch, target_batch) in enumerate(data_loader):
if i < num_batches:
loss = calc_loss_batch(input_batch, target_batch, model, device)
total_loss += loss.item()
else:
break
return total_loss / num_batches
```
> [!TIP]
> **Gradient clipping**은 gradient magnitude에 **최대 임계값**을 설정하여 대규모 neural network의 **training stability**를 향상시키는 데 사용되는 기술입니다. gradient가 미리 정의된 `max_norm`을 초과하면 비례적으로 크기를 줄여 model parameter에 대한 update가 관리 가능한 범위 내에 유지되도록 합니다. 이를 통해 exploding gradients와 같은 문제를 방지하고 더욱 제어되고 안정적인 training을 보장합니다.
>
> _이 개선 사항은 이전 코드에 포함되어 있지 않습니다._

### Training functions

`generate_and_print_sample` 함수는 context를 가져와 몇 개의 token을 생성하며, 해당 시점에서 모델이 얼마나 잘 작동하는지 파악하는 데 사용됩니다. 이 함수는 각 step에서 `train_model_simple`에 의해 호출됩니다.

`evaluate_model` 함수는 training 함수에 지정된 빈도로 호출되며, 해당 모델 training 시점의 train loss와 validation loss를 측정하는 데 사용됩니다.

그리고 실제로 모델을 train하는 큰 함수가 `train_model_simple`입니다. 이 함수는 다음을 필요로 합니다.

- train data loader (데이터가 이미 분리되어 training에 맞게 준비된 상태)
- validator loader
- training 중 사용할 **optimizer**: gradient를 사용하고 loss를 줄이기 위해 parameter를 업데이트하는 함수입니다. 이 경우에는 아래에서 보게 될 `AdamW`를 사용하지만, 그 외에도 다양한 optimizer가 있습니다.
- 각 round에서 gradient가 누적되지 않도록 초기화하기 위해 `optimizer.zero_grad()`를 호출합니다.
- **`lr`** parameter는 모델의 parameter를 업데이트하는 optimization process에서 수행되는 **step의 크기**를 결정하는 **learning rate**입니다. **더 작은** learning rate는 optimizer가 weight를 **더 작게 업데이트**하도록 하며, 더 **정밀한** convergence로 이어질 수 있지만 training이 **느려질** 수 있습니다. **더 큰** learning rate는 training을 빠르게 할 수 있지만 loss function의 minimum을 **지나칠 위험**이 있습니다(즉, loss function이 최소화되는 지점을 **뛰어넘는** 것입니다).
- **Weight Decay**는 큰 weight에 불이익을 주는 추가 항을 더해 **Loss Calculation** 단계를 수정합니다. 이를 통해 optimizer는 더 작은 weight를 사용하는 solution을 찾게 되며, data를 잘 fitting하는 것과 모델을 단순하게 유지하는 것 사이에서 균형을 맞춥니다. 또한 모델이 특정 feature 하나에 지나치게 큰 중요도를 할당하지 않도록 하여 machine learning model의 overfitting을 방지합니다.
- SGD와 L2 regularization 같은 전통적인 optimizer는 weight decay를 loss function의 gradient와 결합합니다. 하지만 **AdamW**(Adam optimizer의 variant)는 weight decay를 gradient update와 분리하여 더 효과적인 regularization을 수행합니다.
- training에 사용할 device
- epoch 수: training data를 반복해서 처리할 횟수
- evaluation frequency: `evaluate_model`을 호출할 빈도
- evaluation iteration: `generate_and_print_sample`을 호출할 때 모델의 현재 상태를 평가하는 데 사용할 batch 수
- start context: `generate_and_print_sample`을 호출할 때 사용할 시작 문장
- tokenizer
```python
# Functions to train the data
def train_model_simple(model, train_loader, val_loader, optimizer, device, num_epochs,
eval_freq, eval_iter, start_context, tokenizer):
# Initialize lists to track losses and tokens seen
train_losses, val_losses, track_tokens_seen = [], [], []
tokens_seen, global_step = 0, -1

# Main training loop
for epoch in range(num_epochs):
model.train()  # Set model to training mode

for input_batch, target_batch in train_loader:
optimizer.zero_grad() # Reset loss gradients from previous batch iteration
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss.backward() # Calculate loss gradients
optimizer.step() # Update model weights using loss gradients
tokens_seen += input_batch.numel()
global_step += 1

# Optional evaluation step
if global_step % eval_freq == 0:
train_loss, val_loss = evaluate_model(
model, train_loader, val_loader, device, eval_iter)
train_losses.append(train_loss)
val_losses.append(val_loss)
track_tokens_seen.append(tokens_seen)
print(f"Ep {epoch+1} (Step {global_step:06d}): "
f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")

# Print a sample text after each epoch
generate_and_print_sample(
model, tokenizer, device, start_context
)

return train_losses, val_losses, track_tokens_seen


def evaluate_model(model, train_loader, val_loader, device, eval_iter):
model.eval() # Set in eval mode to avoid dropout
with torch.no_grad():
train_loss = calc_loss_loader(train_loader, model, device, num_batches=eval_iter)
val_loss = calc_loss_loader(val_loader, model, device, num_batches=eval_iter)
model.train() # Back to training model applying all the configurations
return train_loss, val_loss


def generate_and_print_sample(model, tokenizer, device, start_context):
model.eval() # Set in eval mode to avoid dropout
context_size = model.pos_emb.weight.shape[0]
encoded = text_to_token_ids(start_context, tokenizer).to(device)
with torch.no_grad():
token_ids = generate_text(
model=model, idx=encoded,
max_new_tokens=50, context_size=context_size
)
decoded_text = token_ids_to_text(token_ids, tokenizer)
print(decoded_text.replace("\n", " "))  # Compact print format
model.train() # Back to training model applying all the configurations
```
> [!TIP]
> learning rate를 개선하기 위해 **linear warmup**과 **cosine decay**라는 몇 가지 관련 기법을 사용할 수 있습니다.
>
> **Linear warmup**은 초기 learning rate와 최대 learning rate를 정의하고, 각 epoch 후에 이를 일관되게 업데이트하는 방식입니다. 이는 더 작은 weight update로 학습을 시작하면 학습 단계에서 모델이 크고 불안정한 update를 겪을 위험이 감소하기 때문입니다.\
> **Cosine decay**는 **warmup** 단계 이후 반 cosine 곡선을 따라 **learning rate를 점진적으로 감소**시키는 기법으로, weight update를 늦춰 loss 최솟값을 **지나치는 위험을 최소화**하고 이후 단계에서 학습 안정성을 보장합니다.
>
> _이러한 개선 사항은 이전 코드에 포함되어 있지 않다는 점에 유의하세요._

### 실용적인 최신 training 개선 사항

이전 loop만으로도 기본 메커니즘을 이해할 수 있지만, 최신 pre-training에서는 일반적으로 throughput과 안정성을 향상하기 위해 다음과 같은 최적화를 추가합니다:

- **Mixed precision (`bfloat16` / `float16`)**을 사용해 메모리 사용량을 줄이고 행렬 곱셈을 가속합니다. 실제로 최근 NVIDIA datacenter GPU에서는 더 넓은 지수 범위를 유지하고 일반적으로 gradient scaling이 필요하지 않기 때문에 `bfloat16`이 선호됩니다. 반면 `float16`은 보통 `GradScaler`를 사용합니다.
- **Gradient accumulation**을 사용해 전체 batch가 VRAM에 들어가지 않을 때 더 큰 global batch size를 시뮬레이션합니다.
- 특히 training 초기 단계에서 exploding updates를 방지하기 위해 **Gradient clipping**(예: `clip_grad_norm_`)을 적용합니다.
- **Activation checkpointing**을 사용해 forward pass의 일부를 backpropagation 중 다시 계산하는 대신 메모리 사용량을 크게 줄이고 추가 연산을 감수합니다.
- **Fused / memory-efficient attention**(`scaled_dot_product_attention`)을 사용해 더 긴 context window에서 attention 비용을 줄입니다.
- model 또는 optimizer states가 더 이상 단일 GPU에 들어가지 않을 때 **Sharded training/checkpointing**(예: FSDP / distributed checkpoints)을 사용합니다.

이러한 아이디어 중 일부를 혼합한 PyTorch의 간단한 예시는 다음과 같습니다:
```python
accum_steps = 8
scaler = torch.amp.GradScaler("cuda")
optimizer.zero_grad(set_to_none=True)

for step, (input_batch, target_batch) in enumerate(train_loader, start=1):
with torch.autocast(device_type="cuda", dtype=torch.float16):
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss = loss / accum_steps

scaler.scale(loss).backward()

if step % accum_steps == 0:
scaler.unscale_(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
scaler.step(optimizer)
scaler.update()
optimizer.zero_grad(set_to_none=True)
```
> [!WARNING]
> 체크포인트 또는 third-party repository에서 pre-training할 때는 **로딩 경로를 attack surface의 일부로 취급**하세요. 전체 pickled Python object 대신 `state_dict` 또는 `safetensors` 파일을 로딩하는 것을 우선하고, repository를 감사하지 않았다면 `trust_remote_code=False`를 유지하며, model-loading RCE 예시는 [../AI-Models-RCE.md](../AI-Models-RCE.md)에서 확인하세요.

### 더 안전하고 메모리 효율적인 checkpoint 로딩

대규모 checkpoint의 경우, 단순한 `torch.load(...)` + `load_state_dict(...)` 순서는 메모리를 일시적으로 중복할 수 있습니다(초기화된 parameters + 로딩된 weights). 최신 PyTorch 버전에는 매우 유용한 몇 가지 옵션이 추가되었습니다. PyTorch 2.6부터는 사용자 지정 pickle module을 전달하지 않을 때 `torch.load()`의 기본값이 `weights_only=True`이지만, 예제에서는 명시적으로 지정하는 것이 여전히 좋습니다.

- `weights_only=True`: 임의의 pickled Python objects 대신 tensors / primitive objects로 deserialization을 제한합니다.
- `mmap=True`: 모든 내용을 RAM으로 즉시 읽는 대신 디스크의 tensor storages를 memory-map합니다.
- `with torch.device("meta"):`: 먼저 빈 tensors로 module을 생성합니다.
- `load_state_dict(..., assign=True)`: 로딩된 tensors를 meta로 초기화된 module에 직접 할당합니다.
```python
state_dict = torch.load(
"model.pth",
map_location="cpu",
weights_only=True,
mmap=True,
)

with torch.device("meta"):
model = GPTModel(GPT_CONFIG_124M)

model.load_state_dict(state_dict, assign=True)
model.to(device)
model.eval()
```
이는 multi-GB checkpoint를 workstation에서 로드할 때 특히 유용합니다. 시작 중 RAM 급증을 줄여 주기 때문입니다.

## GPT2 weights 로드

GPT2 weights를 로컬에서 로드하는 2가지 간단한 script가 있습니다. 두 경우 모두 repository [https://github.com/rasbt/LLMs-from-scratch](https://github.com/rasbt/LLMs-from-scratch)를 로컬에 clone한 다음 다음을 수행할 수 있습니다.

- script [https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/01_main-chapter-code/gpt_generate.py](https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/01_main-chapter-code/gpt_generate.py)는 모든 weights를 다운로드하고, OpenAI 형식에서 해당 LLM이 요구하는 형식으로 변환합니다. 또한 필요한 configuration과 `"Every effort moves you"` prompt가 미리 설정되어 있습니다.
- script [https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/02_alternative_weight_loading/weight-loading-hf-transformers.ipynb](https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/02_alternative_weight_loading/weight-loading-hf-transformers.ipynb)를 사용하면 GPT2 weights 중 원하는 것을 로컬에 로드할 수 있으며(`CHOOSE_MODEL` var만 변경), 일부 prompt를 사용해 text를 예측할 수 있습니다.

GPT-2 대신 Hugging Face의 더 최신 open-source checkpoint를 로드하려는 경우, 일반적인 패턴은 `transformers`가 model을 자동으로 배치하도록 하고, 사용 가능한 경우 `safetensors` weights를 우선 사용하는 것입니다:
```python
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

model_id = "google/gemma-7b"

tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(
model_id,
torch_dtype=torch.bfloat16,
device_map="auto",
low_cpu_mem_usage=True,
trust_remote_code=False,
)
```
`device_map="auto"`는 전체 checkpoint가 하나의 accelerator에 들어가지 않을 때 매우 유용합니다. loader가 GPU / CPU 메모리에 레이어를 분산할 수 있기 때문입니다. 대규모 repository의 경우 `save_pretrained(..., max_shard_size="5GB")` 또는 이에 상응하는 Hub checkpoint가 **sharded checkpoint**와 index를 생성하므로, RAM에 하나의 거대한 파일을 요구하지 않고 점진적으로 로드할 수 있습니다.

전체 파일을 로드하지 않고 tensor를 검사하거나 일부를 추출해야 하는 경우 `safetensors`는 partial read도 지원합니다:
```python
from safetensors import safe_open

with safe_open("model.safetensors", framework="pt", device="cpu") as f:
emb_slice = f.get_slice("model.embed_tokens.weight")
vocab_size, hidden_dim = emb_slice.get_shape()
first_half = emb_slice[:, : hidden_dim // 2]
```
