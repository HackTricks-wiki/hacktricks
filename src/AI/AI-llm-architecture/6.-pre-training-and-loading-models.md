## Παραγωγή κειμένου

Για να εκπαιδεύσουμε ένα μοντέλο, θα χρειαστεί να μπορεί να παράγει νέα tokens. Στη συνέχεια, θα συγκρίνουμε τα παραγόμενα tokens με τα αναμενόμενα, ώστε να εκπαιδεύσουμε το μοντέλο να **μαθαίνει τα tokens που χρειάζεται να παράγει**.

Όπως και στα προηγούμενα παραδείγματα, έχουμε ήδη προβλέψει ορισμένα tokens, επομένως είναι δυνατό να επαναχρησιμοποιήσουμε αυτή τη συνάρτηση για αυτόν τον σκοπό.

> [!TIP]
> Ο στόχος αυτής της έκτης φάσης είναι πολύ απλός: **Εκπαίδευση του μοντέλου από το μηδέν**. Για αυτόν τον σκοπό, θα χρησιμοποιηθεί η προηγούμενη αρχιτεκτονική LLM, με ορισμένους βρόχους που θα διατρέχουν τα σύνολα δεδομένων, χρησιμοποιώντας τις καθορισμένες loss functions και τον optimizer για την εκπαίδευση όλων των παραμέτρων του μοντέλου.

## Αξιολόγηση κειμένου

Για να πραγματοποιηθεί σωστή εκπαίδευση, είναι απαραίτητο να μετρώνται και να ελέγχονται οι προβλέψεις που λαμβάνονται για το αναμενόμενο token. Στόχος της εκπαίδευσης είναι η μεγιστοποίηση της πιθανότητας του σωστού token, κάτι που περιλαμβάνει την αύξηση της πιθανότητάς του σε σχέση με τα άλλα tokens.

Για να μεγιστοποιηθεί η πιθανότητα του σωστού token, τα βάρη του μοντέλου πρέπει να τροποποιηθούν, ώστε αυτή η πιθανότητα να μεγιστοποιηθεί. Η ενημέρωση των βαρών πραγματοποιείται μέσω **backpropagation**. Αυτό απαιτεί μια **loss function για μεγιστοποίηση**. Σε αυτήν την περίπτωση, η συνάρτηση θα είναι η **διαφορά μεταξύ της πραγματοποιηθείσας πρόβλεψης και της επιθυμητής**.

Ωστόσο, αντί να εργαζόμαστε με τις ακατέργαστες προβλέψεις, θα χρησιμοποιηθεί λογάριθμος με βάση n. Έτσι, αν η τρέχουσα πρόβλεψη του αναμενόμενου token ήταν 7.4541e-05, ο φυσικός λογάριθμος (βάση *e*) του **7.4541e-05** είναι περίπου **-9.5042**.\
Στη συνέχεια, για κάθε καταχώριση με context length 5 tokens, για παράδειγμα, το μοντέλο θα πρέπει να προβλέψει 5 tokens, όπου τα πρώτα 4 tokens είναι το τελευταίο token της εισόδου και το πέμπτο είναι το προβλεπόμενο. Επομένως, για κάθε καταχώριση θα έχουμε 5 προβλέψεις σε αυτήν την περίπτωση (ακόμη και αν οι 4 πρώτες βρίσκονταν στην είσοδο, το μοντέλο δεν το γνωρίζει), με 5 αναμενόμενα tokens και επομένως 5 πιθανότητες προς μεγιστοποίηση.

Επομένως, αφού υπολογιστεί ο φυσικός λογάριθμος για κάθε πρόβλεψη, υπολογίζεται ο **μέσος όρος**, αφαιρείται το **σύμβολο του μείον** (αυτό ονομάζεται _cross entropy loss_) και αυτός είναι ο **αριθμός που πρέπει να μειωθεί όσο το δυνατόν πιο κοντά στο 0**, επειδή ο φυσικός λογάριθμος του 1 είναι 0:

Ένας ακόμη τρόπος μέτρησης του πόσο καλό είναι το μοντέλο ονομάζεται perplexity. Η **perplexity** είναι ένα metric που χρησιμοποιείται για την αξιολόγηση του πόσο καλά ένα probability model προβλέπει ένα δείγμα. Στο language modelling, αναπαριστά την **αβεβαιότητα του μοντέλου** κατά την πρόβλεψη του επόμενου token σε μια ακολουθία.\
Για παράδειγμα, μια τιμή perplexity ίση με 48725 σημαίνει ότι, όταν πρέπει να προβλέψει ένα token, το μοντέλο δεν είναι βέβαιο για το ποιο από τα 48.725 tokens του vocabulary είναι το σωστό.

```python
import tiktoken
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader


class GPTDatasetV1(Dataset):
def __init__(self, txt, tokenizer, max_length, stride):
self.input_ids = []
self.target_ids = []

# Tokenize the entire text
token_ids = tokenizer.encode(txt, allowed_special={"<|endoftext|>"})

# Use a sliding window to chunk the book into overlapping sequences of max_length
for i in range(0, len(token_ids) - max_length, stride):
input_chunk = token_ids[i:i + max_length]
target_chunk = token_ids[i + 1: i + max_length + 1]
self.input_ids.append(torch.tensor(input_chunk))
self.target_ids.append(torch.tensor(target_chunk))

def __len__(self):
return len(self.input_ids)

def __getitem__(self, idx):
return self.input_ids[idx], self.target_ids[idx]


def create_dataloader_v1(txt, batch_size=4, max_length=256,
stride=128, shuffle=True, drop_last=True, num_workers=0):
# Initialize the tokenizer
tokenizer = tiktoken.get_encoding("gpt2")

# Create dataset
dataset = GPTDatasetV1(txt, tokenizer, max_length, stride)

# Create dataloader
dataloader = DataLoader(
dataset, batch_size=batch_size, shuffle=shuffle, drop_last=drop_last, num_workers=num_workers)

return dataloader


class MultiHeadAttention(nn.Module):
def __init__(self, d_in, d_out, context_length, dropout, num_heads, qkv_bias=False):
super().__init__()
assert d_out % num_heads == 0, "d_out must be divisible by n_heads"

self.d_out = d_out
self.num_heads = num_heads
self.head_dim = d_out // num_heads  # Reduce the projection dim to match desired output dim

self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_key = nn.Linear(d_in, d_out, bias=qkv_bias)
self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
self.out_proj = nn.Linear(d_out, d_out)  # Linear layer to combine head outputs
self.dropout = nn.Dropout(dropout)
self.register_buffer('mask', torch.triu(torch.ones(context_length, context_length), diagonal=1))

def forward(self, x):
b, num_tokens, d_in = x.shape

keys = self.W_key(x)  # Shape: (b, num_tokens, d_out)
queries = self.W_query(x)
values = self.W_value(x)

# We implicitly split the matrix by adding a `num_heads` dimension
# Unroll last dim: (b, num_tokens, d_out) -> (b, num_tokens, num_heads, head_dim)
keys = keys.view(b, num_tokens, self.num_heads, self.head_dim)
values = values.view(b, num_tokens, self.num_heads, self.head_dim)
queries = queries.view(b, num_tokens, self.num_heads, self.head_dim)

# Transpose: (b, num_tokens, num_heads, head_dim) -> (b, num_heads, num_tokens, head_dim)
keys = keys.transpose(1, 2)
queries = queries.transpose(1, 2)
values = values.transpose(1, 2)

# Compute scaled dot-product attention (aka self-attention) with a causal mask
attn_scores = queries @ keys.transpose(2, 3)  # Dot product for each head

# Original mask truncated to the number of tokens and converted to boolean
mask_bool = self.mask.bool()[:num_tokens, :num_tokens]

# Use the mask to fill attention scores
attn_scores.masked_fill_(mask_bool, -torch.inf)

attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
attn_weights = self.dropout(attn_weights)

# Shape: (b, num_tokens, num_heads, head_dim)
context_vec = (attn_weights @ values).transpose(1, 2)

# Combine heads, where self.d_out = self.num_heads * self.head_dim
context_vec = context_vec.reshape(b, num_tokens, self.d_out)
context_vec = self.out_proj(context_vec)  # optional projection

return context_vec


class LayerNorm(nn.Module):
def __init__(self, emb_dim):
super().__init__()
self.eps = 1e-5
self.scale = nn.Parameter(torch.ones(emb_dim))
self.shift = nn.Parameter(torch.zeros(emb_dim))

def forward(self, x):
mean = x.mean(dim=-1, keepdim=True)
var = x.var(dim=-1, keepdim=True, unbiased=False)
norm_x = (x - mean) / torch.sqrt(var + self.eps)
return self.scale * norm_x + self.shift


class GELU(nn.Module):
def __init__(self):
super().__init__()

def forward(self, x):
return 0.5 * x * (1 + torch.tanh(
torch.sqrt(torch.tensor(2.0 / torch.pi)) *
(x + 0.044715 * torch.pow(x, 3))
))


class FeedForward(nn.Module):
def __init__(self, cfg):
super().__init__()
self.layers = nn.Sequential(
nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),
GELU(),
nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"]),
)

def forward(self, x):
return self.layers(x)


class TransformerBlock(nn.Module):
def __init__(self, cfg):
super().__init__()
self.att = MultiHeadAttention(
d_in=cfg["emb_dim"],
d_out=cfg["emb_dim"],
context_length=cfg["context_length"],
num_heads=cfg["n_heads"],
dropout=cfg["drop_rate"],
qkv_bias=cfg["qkv_bias"])
self.ff = FeedForward(cfg)
self.norm1 = LayerNorm(cfg["emb_dim"])
self.norm2 = LayerNorm(cfg["emb_dim"])
self.drop_shortcut = nn.Dropout(cfg["drop_rate"])

def forward(self, x):
# Shortcut connection for attention block
shortcut = x
x = self.norm1(x)
x = self.att(x)   # Shape [batch_size, num_tokens, emb_size]
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

# Shortcut connection for feed-forward block
shortcut = x
x = self.norm2(x)
x = self.ff(x)
x = self.drop_shortcut(x)
x = x + shortcut  # Add the original input back

return x


class GPTModel(nn.Module):
def __init__(self, cfg):
super().__init__()
self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])
self.drop_emb = nn.Dropout(cfg["drop_rate"])

self.trf_blocks = nn.Sequential(
*[TransformerBlock(cfg) for _ in range(cfg["n_layers"])])

self.final_norm = LayerNorm(cfg["emb_dim"])
self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

def forward(self, in_idx):
batch_size, seq_len = in_idx.shape
tok_embeds = self.tok_emb(in_idx)
pos_embeds = self.pos_emb(torch.arange(seq_len, device=in_idx.device))
x = tok_embeds + pos_embeds  # Shape [batch_size, num_tokens, emb_size]
x = self.drop_emb(x)
x = self.trf_blocks(x)
x = self.final_norm(x)
logits = self.out_head(x)
return logits
```

### Συναρτήσεις Generate text

Σε προηγούμενη ενότητα παρουσιάστηκε μια συνάρτηση που επέστρεφε απλώς το **πιο πιθανό token** μετά τη λήψη των logits. Ωστόσο, αυτό σημαίνει ότι για κάθε είσοδο θα παράγεται πάντα η ίδια έξοδος, γεγονός που την καθιστά πολύ deterministic.

Η ακόλουθη συνάρτηση `generate_text` εφαρμόζει τις έννοιες των `top-k`, `temperature` και `multinomial`.

- Το **`top-k`** σημαίνει ότι θα αρχίσουμε να μειώνουμε σε `-inf` όλες τις πιθανότητες όλων των token εκτός από τα k κορυφαία token. Έτσι, αν k=3, πριν από τη λήψη απόφασης μόνο τα 3 πιο πιθανά token θα έχουν πιθανότητα διαφορετική από `-inf`.
- Το **`temperature`** σημαίνει ότι κάθε πιθανότητα θα διαιρείται με την τιμή του temperature. Μια τιμή `0.1` θα ενισχύσει την υψηλότερη πιθανότητα σε σύγκριση με τη χαμηλότερη, ενώ ένα temperature ίσο με `5`, για παράδειγμα, θα κάνει την κατανομή πιο επίπεδη. Αυτό βοηθά στη βελτίωση της ποικιλίας των responses που θέλουμε να έχει το LLM.
- Μετά την εφαρμογή του temperature, εφαρμόζεται ξανά μια συνάρτηση **`softmax`**, ώστε όλα τα εναπομείναντα token να έχουν συνολική πιθανότητα ίση με 1.
- Τέλος, αντί να επιλεγεί το token με τη μεγαλύτερη πιθανότητα, εφαρμόζεται η συνάρτηση **`multinomial`** για να **προβλέψει το επόμενο token σύμφωνα με τις τελικές πιθανότητες**. Έτσι, αν το token 1 έχει πιθανότητα 70%, το token 2 πιθανότητα 20% και το token 3 πιθανότητα 10%, στο 70% των περιπτώσεων θα επιλέγεται το token 1, στο 20% των περιπτώσεων το token 2 και στο 10% των περιπτώσεων το token 3.
```python
# Generate text function
def generate_text(model, idx, max_new_tokens, context_size, temperature=0.0, top_k=None, eos_id=None):

# For-loop is the same as before: Get logits, and only focus on last time step
for _ in range(max_new_tokens):
idx_cond = idx[:, -context_size:]
with torch.no_grad():
logits = model(idx_cond)
logits = logits[:, -1, :]

# New: Filter logits with top_k sampling
if top_k is not None:
# Keep only top_k values
top_logits, _ = torch.topk(logits, top_k)
min_val = top_logits[:, -1]
logits = torch.where(logits < min_val, torch.tensor(float("-inf")).to(logits.device), logits)

# New: Apply temperature scaling
if temperature > 0.0:
logits = logits / temperature

# Apply softmax to get probabilities
probs = torch.softmax(logits, dim=-1)  # (batch_size, context_len)

# Sample from the distribution
idx_next = torch.multinomial(probs, num_samples=1)  # (batch_size, 1)

# Otherwise same as before: get idx of the vocab entry with the highest logits value
else:
idx_next = torch.argmax(logits, dim=-1, keepdim=True)  # (batch_size, 1)

if idx_next == eos_id:  # Stop generating early if end-of-sequence token is encountered and eos_id is specified
break

# Same as before: append sampled index to the running sequence
idx = torch.cat((idx, idx_next), dim=1)  # (batch_size, num_tokens+1)

return idx
```
> [!TIP]
> Υπάρχει μια κοινή εναλλακτική του `top-k` που ονομάζεται [**`top-p`**](https://en.wikipedia.org/wiki/Top-p_sampling), γνωστή επίσης ως nucleus sampling, η οποία, αντί να λαμβάνει k δείγματα με τη μεγαλύτερη πιθανότητα, **οργανώνει** ολόκληρο το **λεξιλόγιο** βάσει πιθανοτήτων και τις **αθροίζει** από τη μεγαλύτερη πιθανότητα προς τη μικρότερη, μέχρι να **επιτευχθεί ένα κατώφλι**.
>
> Στη συνέχεια, **μόνο αυτές οι λέξεις** του λεξιλογίου θα λαμβάνονται υπόψη σύμφωνα με τις σχετικές πιθανότητές τους
>
> Αυτό επιτρέπει να μη χρειάζεται η επιλογή ενός αριθμού `k` δειγμάτων, καθώς το βέλτιστο k μπορεί να διαφέρει σε κάθε περίπτωση, αλλά να απαιτείται **μόνο ένα κατώφλι**.
>
> _Σημειώστε ότι αυτή η βελτίωση δεν περιλαμβάνεται στον προηγούμενο κώδικα._

> [!TIP]
> Ένας ακόμη τρόπος βελτίωσης του παραγόμενου κειμένου είναι η χρήση του **Beam search** αντί για το greedy search που χρησιμοποιείται σε αυτό το παράδειγμα.\
> Σε αντίθεση με το greedy search, το οποίο επιλέγει την πιο πιθανή επόμενη λέξη σε κάθε βήμα και δημιουργεί μία μόνο ακολουθία, το **beam search διατηρεί τις 𝑘 k ακολουθίες με την υψηλότερη βαθμολογία** (οι οποίες ονομάζονται "beams") σε κάθε βήμα. Εξερευνώντας ταυτόχρονα πολλαπλές δυνατότητες, εξισορροπεί την αποδοτικότητα και την ποιότητα, αυξάνοντας τις πιθανότητες **εύρεσης μιας συνολικά καλύτερης** ακολουθίας, η οποία μπορεί να παραβλεφθεί από την προσέγγιση greedy λόγω πρώιμων, μη βέλτιστων επιλογών.
>
> _Σημειώστε ότι αυτή η βελτίωση δεν περιλαμβάνεται στον προηγούμενο κώδικα._

### Συναρτήσεις απώλειας

Η συνάρτηση **`calc_loss_batch`** υπολογίζει το cross entropy μιας πρόβλεψης για ένα μεμονωμένο batch.\
Η **`calc_loss_loader`** υπολογίζει το cross entropy όλων των batches και υπολογίζει το **μέσο cross entropy**.
```python
# Define loss functions
def calc_loss_batch(input_batch, target_batch, model, device):
input_batch, target_batch = input_batch.to(device), target_batch.to(device)
logits = model(input_batch)
loss = torch.nn.functional.cross_entropy(logits.flatten(0, 1), target_batch.flatten())
return loss

def calc_loss_loader(data_loader, model, device, num_batches=None):
total_loss = 0.
if len(data_loader) == 0:
return float("nan")
elif num_batches is None:
num_batches = len(data_loader)
else:
# Reduce the number of batches to match the total number of batches in the data loader
# if num_batches exceeds the number of batches in the data loader
num_batches = min(num_batches, len(data_loader))
for i, (input_batch, target_batch) in enumerate(data_loader):
if i < num_batches:
loss = calc_loss_batch(input_batch, target_batch, model, device)
total_loss += loss.item()
else:
break
return total_loss / num_batches
```
> [!TIP]
> Το **Gradient clipping** είναι μια τεχνική που χρησιμοποιείται για τη βελτίωση της **σταθερότητας της εκπαίδευσης** σε μεγάλα νευρωνικά δίκτυα, ορίζοντας ένα **μέγιστο όριο** για τα μεγέθη των gradients. Όταν τα gradients υπερβαίνουν το προκαθορισμένο `max_norm`, μειώνονται αναλογικά, ώστε οι ενημερώσεις των παραμέτρων του μοντέλου να παραμένουν σε διαχειρίσιμο εύρος, αποτρέποντας προβλήματα όπως τα exploding gradients και εξασφαλίζοντας πιο ελεγχόμενη και σταθερή εκπαίδευση.
>
> _Σημειώστε ότι αυτή η βελτίωση δεν περιλαμβάνεται στον προηγούμενο κώδικα._

### Συναρτήσεις training

Η συνάρτηση `generate_and_print_sample` απλώς θα λάβει ένα context και θα παράγει μερικά tokens, ώστε να αποκτήσουμε μια εικόνα για το πόσο καλό είναι το model σε εκείνο το σημείο. Καλείται από την `train_model_simple` σε κάθε step.

Η συνάρτηση `evaluate_model` καλείται τόσο συχνά όσο υποδεικνύεται στη συνάρτηση training και χρησιμοποιείται για τη μέτρηση του train loss και του validation loss σε εκείνο το σημείο του model training.

Στη συνέχεια, η μεγάλη συνάρτηση `train_model_simple` είναι αυτή που εκτελεί πραγματικά το training του model. Αναμένει:

- Το train data loader (με τα δεδομένα ήδη διαχωρισμένα και προετοιμασμένα για training)
- Το validator loader
- Το **optimizer** που θα χρησιμοποιηθεί κατά το training: Αυτή είναι η συνάρτηση που θα χρησιμοποιεί τα gradients και θα ενημερώνει τις παραμέτρους για να μειώσει το loss. Σε αυτή την περίπτωση, όπως θα δείτε, χρησιμοποιείται το `AdamW`, αλλά υπάρχουν πολλά ακόμη.
- Η `optimizer.zero_grad()` καλείται για να επαναφέρει τα gradients σε κάθε round, ώστε να μην συσσωρεύονται.
- Η παράμετρος **`lr`** είναι το **learning rate**, το οποίο καθορίζει το **μέγεθος των βημάτων** που εκτελούνται κατά τη διαδικασία optimization, όταν ενημερώνονται οι παράμετροι του model. Ένα **μικρότερο** learning rate σημαίνει ότι ο optimizer **εκτελεί μικρότερες ενημερώσεις** στα weights, κάτι που μπορεί να οδηγήσει σε πιο **ακριβή** σύγκλιση, αλλά ενδέχεται να **επιβραδύνει** το training. Ένα **μεγαλύτερο** learning rate μπορεί να επιταχύνει το training, αλλά υπάρχει κίνδυνος να **ξεπεράσει** το minimum της loss function (**να υπερπηδήσει** το σημείο στο οποίο ελαχιστοποιείται η loss function).
- Το **Weight Decay** τροποποιεί το βήμα **Loss Calculation**, προσθέτοντας έναν επιπλέον όρο που penalizes τα μεγάλα weights. Αυτό ενθαρρύνει τον optimizer να βρίσκει λύσεις με μικρότερα weights, εξισορροπώντας την καλή προσαρμογή στα δεδομένα με τη διατήρηση της απλότητας του model και αποτρέποντας το overfitting στα machine learning models, αποθαρρύνοντας το model από το να αποδίδει υπερβολική σημασία σε οποιοδήποτε μεμονωμένο feature.
- Παραδοσιακοί optimizers, όπως ο SGD με L2 regularization, συνδέουν το weight decay με το gradient της loss function. Ωστόσο, ο **AdamW** (μια παραλλαγή του Adam optimizer) αποσυνδέει το weight decay από το gradient update, οδηγώντας σε αποτελεσματικότερο regularization.
- Το device που θα χρησιμοποιηθεί για το training
- Ο αριθμός των epochs: Ο αριθμός των φορών που θα επαναληφθούν τα training data
- Το evaluation frequency: Η συχνότητα με την οποία θα καλείται η `evaluate_model`
- Το evaluation iteration: Ο αριθμός των batches που θα χρησιμοποιούνται κατά την αξιολόγηση της τρέχουσας κατάστασης του model, όταν καλείται η `generate_and_print_sample`
- Το start context: Η αρχική πρόταση που θα χρησιμοποιείται όταν καλείται η `generate_and_print_sample`
- Ο tokenizer
```python
# Functions to train the data
def train_model_simple(model, train_loader, val_loader, optimizer, device, num_epochs,
eval_freq, eval_iter, start_context, tokenizer):
# Initialize lists to track losses and tokens seen
train_losses, val_losses, track_tokens_seen = [], [], []
tokens_seen, global_step = 0, -1

# Main training loop
for epoch in range(num_epochs):
model.train()  # Set model to training mode

for input_batch, target_batch in train_loader:
optimizer.zero_grad() # Reset loss gradients from previous batch iteration
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss.backward() # Calculate loss gradients
optimizer.step() # Update model weights using loss gradients
tokens_seen += input_batch.numel()
global_step += 1

# Optional evaluation step
if global_step % eval_freq == 0:
train_loss, val_loss = evaluate_model(
model, train_loader, val_loader, device, eval_iter)
train_losses.append(train_loss)
val_losses.append(val_loss)
track_tokens_seen.append(tokens_seen)
print(f"Ep {epoch+1} (Step {global_step:06d}): "
f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")

# Print a sample text after each epoch
generate_and_print_sample(
model, tokenizer, device, start_context
)

return train_losses, val_losses, track_tokens_seen


def evaluate_model(model, train_loader, val_loader, device, eval_iter):
model.eval() # Set in eval mode to avoid dropout
with torch.no_grad():
train_loss = calc_loss_loader(train_loader, model, device, num_batches=eval_iter)
val_loss = calc_loss_loader(val_loader, model, device, num_batches=eval_iter)
model.train() # Back to training model applying all the configurations
return train_loss, val_loss


def generate_and_print_sample(model, tokenizer, device, start_context):
model.eval() # Set in eval mode to avoid dropout
context_size = model.pos_emb.weight.shape[0]
encoded = text_to_token_ids(start_context, tokenizer).to(device)
with torch.no_grad():
token_ids = generate_text(
model=model, idx=encoded,
max_new_tokens=50, context_size=context_size
)
decoded_text = token_ids_to_text(token_ids, tokenizer)
print(decoded_text.replace("\n", " "))  # Compact print format
model.train() # Back to training model applying all the configurations
```
> [!TIP]
> Για τη βελτίωση του learning rate υπάρχουν δύο σχετικές τεχνικές που ονομάζονται **linear warmup** και **cosine decay.**
>
> Το **linear warmup** συνίσταται στον ορισμό ενός αρχικού learning rate και ενός μέγιστου learning rate και στη συνεπή ενημέρωσή του μετά από κάθε epoch. Αυτό συμβαίνει επειδή η έναρξη της εκπαίδευσης με μικρότερες ενημερώσεις βαρών μειώνει τον κίνδυνο το μοντέλο να αντιμετωπίσει μεγάλες, αποσταθεροποιητικές ενημερώσεις κατά τη φάση εκπαίδευσής του.\
> Το **cosine decay** είναι μια τεχνική που **μειώνει σταδιακά το learning rate** ακολουθώντας μια καμπύλη μισού συνημιτόνου **μετά τη φάση warmup**, επιβραδύνοντας τις ενημερώσεις βαρών ώστε να **ελαχιστοποιηθεί ο κίνδυνος υπέρβασης** των ελαχίστων της loss και να διασφαλιστεί η σταθερότητα της εκπαίδευσης στις μεταγενέστερες φάσεις.
>
> _Σημειώστε ότι αυτές οι βελτιώσεις δεν περιλαμβάνονται στον προηγούμενο κώδικα._

Ή απλώς το μοντέλο, αν σκοπεύετε να το χρησιμοποιήσετε:
```python
# Save the model
torch.save(model.state_dict(), "model.pth")

# Load it
model = GPTModel(GPT_CONFIG_124M)

model.load_state_dict(torch.load("model.pth", map_location=device, weights_only=True))

model.eval() # Put in eval mode
```
### Πρακτικές σύγχρονες βελτιώσεις εκπαίδευσης

Ο προηγούμενος βρόχος αρκεί για την κατανόηση των μηχανισμών, αλλά το σύγχρονο pre-training συνήθως προσθέτει αρκετές βελτιστοποιήσεις για τη βελτίωση της απόδοσης και της σταθερότητας:

- **Mixed precision (`bfloat16` / `float16`)** για τη μείωση της κατανάλωσης μνήμης και την επιτάχυνση των πολλαπλασιασμών πινάκων. Στην πράξη, το `bfloat16` προτιμάται συνήθως σε πρόσφατες NVIDIA GPUs κέντρων δεδομένων, επειδή διατηρεί μεγαλύτερο εύρος εκθέτη και συνήθως δεν χρειάζεται gradient scaling, ενώ το `float16` χρησιμοποιεί συνήθως ένα `GradScaler`.
- **Gradient accumulation** για την προσομοίωση μεγαλύτερου global batch size όταν ολόκληρο το batch δεν χωράει στη VRAM.
- **Gradient clipping** (για παράδειγμα `clip_grad_norm_`) για την αποτροπή εκρηκτικών ενημερώσεων, ειδικά στα αρχικά στάδια της εκπαίδευσης.
- **Activation checkpointing** για την ανταλλαγή επιπλέον υπολογιστικού κόστους με πολύ χαμηλότερη χρήση μνήμης, μέσω του επανυπολογισμού μέρους του forward pass κατά το backpropagation.
- **Fused / memory-efficient attention** (`scaled_dot_product_attention`) για τη μείωση του κόστους του attention σε μεγαλύτερα context windows.
- **Sharded training/checkpointing** (για παράδειγμα FSDP / distributed checkpoints) όταν το μοντέλο ή τα optimizer states δεν χωρούν πλέον σε μία μόνο GPU.

Ένα ελάχιστο παράδειγμα που συνδυάζει μερικές από αυτές τις ιδέες στο PyTorch είναι:
```python
accum_steps = 8
scaler = torch.amp.GradScaler("cuda")
optimizer.zero_grad(set_to_none=True)

for step, (input_batch, target_batch) in enumerate(train_loader, start=1):
with torch.autocast(device_type="cuda", dtype=torch.float16):
loss = calc_loss_batch(input_batch, target_batch, model, device)
loss = loss / accum_steps

scaler.scale(loss).backward()

if step % accum_steps == 0:
scaler.unscale_(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
scaler.step(optimizer)
scaler.update()
optimizer.zero_grad(set_to_none=True)
```
> [!WARNING]
> Κατά το pre-training από checkpoints ή third-party repositories, θεωρήστε τη **loading path μέρος του attack surface**. Προτιμήστε τη φόρτωση `state_dict`s ή αρχείων `safetensors` αντί για ολόκληρα pickled Python objects, διατηρήστε το `trust_remote_code=False` εκτός αν έχετε ελέγξει το repository και δείτε το [../AI-Models-RCE.md](../AI-Models-RCE.md) για παραδείγματα RCE κατά το model-loading.

### Ασφαλέστερη και πιο αποδοτική ως προς τη μνήμη φόρτωση checkpoint

Για μεγάλα checkpoints, η αφελής ακολουθία `torch.load(...)` + `load_state_dict(...)` μπορεί προσωρινά να διπλασιάσει τη χρήση μνήμης (αρχικοποιημένες παράμετροι + φορτωμένα βάρη). Οι νεότερες εκδόσεις του PyTorch πρόσθεσαν ορισμένες πολύ χρήσιμες επιλογές. Από το PyTorch 2.6, το `weights_only=True` είναι η προεπιλογή για το `torch.load()` όταν δεν παρέχεται custom pickle module, αλλά εξακολουθεί να είναι καλή πρακτική να το δηλώνετε ρητά στα παραδείγματα:

- `weights_only=True`: περιορίζει το deserialization σε tensors / primitive objects αντί για αυθαίρετα pickled Python objects.
- `mmap=True`: αντιστοιχίζει memory-map τα tensor storages από τον δίσκο αντί να διαβάζει eager όλα τα δεδομένα στη RAM.
- `with torch.device("meta"):`: δημιουργεί πρώτα το module με κενά tensors.
- `load_state_dict(..., assign=True)`: αναθέτει απευθείας τα φορτωμένα tensors στο module που έχει αρχικοποιηθεί με meta.
```python
state_dict = torch.load(
"model.pth",
map_location="cpu",
weights_only=True,
mmap=True,
)

with torch.device("meta"):
model = GPTModel(GPT_CONFIG_124M)

model.load_state_dict(state_dict, assign=True)
model.to(device)
model.eval()
```
Αυτό είναι ιδιαίτερα χρήσιμο κατά τη φόρτωση checkpoint πολλών GB σε έναν σταθμό εργασίας, επειδή μειώνει τις αιχμές χρήσης RAM κατά την εκκίνηση.

## Φόρτωση των βαρών GPT2

Υπάρχουν 2 γρήγορα scripts για την τοπική φόρτωση των βαρών GPT2. Και για τα δύο μπορείτε πρώτα να κάνετε clone το repository [https://github.com/rasbt/LLMs-from-scratch](https://github.com/rasbt/LLMs-from-scratch) τοπικά και στη συνέχεια:

- Το script [https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/01_main-chapter-code/gpt_generate.py](https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/01_main-chapter-code/gpt_generate.py) θα κατεβάσει όλα τα βάρη και θα μετατρέψει τις μορφές από OpenAI σε εκείνες που αναμένει το LLM μας. Το script είναι επίσης έτοιμο με την απαιτούμενη διαμόρφωση και με το prompt: "Every effort moves you"
- Το script [https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/02_alternative_weight_loading/weight-loading-hf-transformers.ipynb](https://github.com/rasbt/LLMs-from-scratch/blob/main/ch05/02_alternative_weight_loading/weight-loading-hf-transformers.ipynb) σας επιτρέπει να φορτώσετε τοπικά οποιοδήποτε από τα βάρη GPT2 (απλώς αλλάξτε τη μεταβλητή `CHOOSE_MODEL`) και να προβλέψετε κείμενο από ορισμένα prompts.

Αν, αντί για GPT-2, θέλετε να φορτώσετε ένα πιο σύγχρονο open-source checkpoint από το Hugging Face, η συνήθης πρακτική είναι να αφήσετε το `transformers` να τοποθετήσει αυτόματα το μοντέλο και να προτιμήσετε βάρη `safetensors` όταν είναι διαθέσιμα:
```python
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

model_id = "google/gemma-7b"

tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(
model_id,
torch_dtype=torch.bfloat16,
device_map="auto",
low_cpu_mem_usage=True,
trust_remote_code=False,
)
```
Το `device_map="auto"` είναι πολύ χρήσιμο όταν ολόκληρο το checkpoint δεν χωράει σε έναν accelerator, επειδή ο loader μπορεί να κατανείμει τα layers στη μνήμη GPU / CPU. Για μεγάλα repositories, τα `save_pretrained(..., max_shard_size="5GB")` ή τα αντίστοιχα Hub checkpoints δημιουργούν **sharded checkpoints** μαζί με ένα index, επιτρέποντας τη σταδιακή φόρτωσή τους αντί να απαιτείται ένα τεράστιο αρχείο στη RAM.

Αν χρειάζεται να επιθεωρήσετε ή να τεμαχίσετε tensors χωρίς να φορτώσετε ολόκληρο το αρχείο, το `safetensors` υποστηρίζει επίσης partial reads:
```python
from safetensors import safe_open

with safe_open("model.safetensors", framework="pt", device="cpu") as f:
emb_slice = f.get_slice("model.embed_tokens.weight")
vocab_size, hidden_dim = emb_slice.get_shape()
first_half = emb_slice[:, : hidden_dim // 2]
```
